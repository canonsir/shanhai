## 1. 交付 ① — `ObservationDraft` Contract（冻结）

### 1.1 定义原则：`ObservationDraft` = spine `Observation` **去掉身份**

`ObservationDraft` 与读侧投影 [`Observation`](../../services/market-data/shanhai_market_data/ports/observation_reader.py#L37) **同构，唯独没有身份字段**。这是「provider 不背身份职责、身份只在共享层定」（ADR 0021 D4/D4a）的字段级落点：

```
读侧 Observation（spine 一行，带身份）        写侧 ObservationDraft（provider 产出，无身份）
──────────────────────────────────────      ──────────────────────────────────────────
logical_key      ← 共享 ingestion 定          （无）  ← provider 绝不产出
content_hash     ← 共享 ingestion 定          （无）  ← provider 绝不产出
fact_type                                     fact_type          ✅
subject                                        subject            ✅
predicate                                      predicate          ✅
object_value                                   object_value       ✅
occurred_at                                    occurred_at        ✅
published_at                                   published_at       ✅
captured_at                                    captured_at        ✅
confidence (= 1.0 默认)                        （不在 draft，见 §1.4 裁决）
source_ref                                     source_ref         ✅
```

### 1.2 冻结字段（provider MUST 产出的语义草稿）

| 字段 | 类型 | 必填 | 语义 | 对齐既有类型 |
|--|--|--|--|--|
| `subject` | `SubjectRef` | ✅ | 观测对象（`entity_type` / `entity_id` / `label?`） | 复用 [models.SubjectRef](../../services/market-data/shanhai_market_data/models.py#L224) |
| `fact_type` | `FactType` | ✅ | 事实家族：`quote/financial/announcement/news/industry/profile/policy/anomaly/capital_flow/shareholder` | 复用 [models.FactType](../../services/market-data/shanhai_market_data/models.py#L54) |
| `predicate` | `str \| None` | 可选 | 细分语义键（如 `"valuation.pe_ttm"`）；进入 `logical_key` 计算 | 同 `Observation.predicate` |
| `object_value` | `str \| None` | 可选 | 观测值（字符串承载，跨源统一；结构化明细走 typed detail 表） | 同 `Observation.object_value` |
| `occurred_at` | `datetime \| None` | 可选 | effective time — 事件发生 | 同 `Observation.occurred_at` |
| `published_at` | `datetime \| None` | 可选 | effective time — 市场可见 | 同 `Observation.published_at` |
| `captured_at` | `datetime` | ✅ | knowledge time — provider 取到此观测的时刻 | 同 `Observation.captured_at` |
| `source_ref` | `SourceRef` | ✅ | provenance；**provider 身份只活在这里** | 复用 [models.SourceRef](../../services/market-data/shanhai_market_data/models.py#L94) |

- 契约形态：`_FrozenModel`（`frozen=True, extra="forbid"`）；集合字段用 `tuple`（沿用既有约定）。
- **`object_value` 用字符串跨源统一**（沿用 `Observation.object_value: str | None`）；iFinD/Wind 富字段、结构化明细经 typed detail 表投影，**不塞 draft 顶层**（ADR 0021 D5）。

### 1.3 冻结禁止字段（provider MUST NOT 产出）

```
❌ logical_key        —— observation 身份，共享 ingestion 确定性算（D4）
❌ content_hash       —— observation 内容哈希，共享 ingestion 确定性算（D4）
❌ observation_id     —— 任何 observation 级主键/代理键（落库时由 spine/ingestion 负责）
❌ version            —— observation 版本号（append-only spine 的版本语义由身份链表达，非 provider 设）
❌ knowledge 字段     —— Belief / KnowledgeObject / KnowledgeRevision / KnowledgeView（D6-2；provider 不产 Knowledge）
❌ confidence         —— 不在 draft 顶层（§1.4 裁决）
```

### 1.4 关键精度澄清（Contract Review 必须钉死，否则契约自相矛盾）

**(a) `SourceRef.version` / `SourceRef.hash` ≠ 被禁的 `version` / `content_hash`。**
既有 [`SourceRef`](../../services/market-data/shanhai_market_data/models.py#L94) 自身带 `version: str = "v1"`（adapter 版本）和 `hash: str | None`（**源快照**内容哈希）。这两个是 **source snapshot 的 provenance**，描述「这批源数据从哪来、什么版本的 adapter 抓的」，**允许**且**应当**经 `source_ref` 携带。

被 §1.3 禁止的 `version` / `content_hash` 指的是 **observation 自身的身份**——即读侧 `Observation.logical_key` / `Observation.content_hash`。二者层级不同：

```
source_ref.hash / source_ref.version  →  「源快照」的 provenance（provider 填，合法）
Observation.content_hash / logical_key →  「观测在 spine 的身份」（共享 ingestion 定，provider 禁碰）
```

provider 填 `source_ref.hash`（源快照哈希）**不等于**自算 observation 身份——身份是 ingestion 对 draft 语义值确定性哈希得出的，与源快照哈希是两码事。

**(b) `confidence` 裁决：不进 draft 顶层，M3.6.0 定。**
既有 spine `Observation` 有 `confidence: float = 1.0`。裁决：**`ObservationDraft` 不含 `confidence`**。理由：

- provider 表达「源可信度」的正确通道是 `source_ref.trust_level`（`L1_official … L4_derived`，已在 `SourceRef`）——这是**源级**信任，provider 天然知道。
- 一个独立的 observation 级 `confidence` 标量若由 provider 设，容易滑向「provider 做判断/打分」——那是 Knowledge Evolution / Reasoning 的职责（D6-2/3），不是搬运工职责。
- ingestion `_materialize(draft) → Observation` 时，`confidence` 取 `Observation` 模型默认值 `1.0`（ingestion 不计算它，只沿用模型默认，不违反「共享层只定身份」）。
- 若未来某 provider 确需 per-observation confidence，另经 Review 扩契约，不在 M3.6.0 预开口子。

### 1.5 契约草图（sketch，**非实现**）

```python
# contract sketch —— 非实现；M3.6.1 落码。frozen=True, extra="forbid"
class ObservationDraft(_FrozenModel):
    subject: SubjectRef
    fact_type: FactType
    predicate: str | None = None
    object_value: str | None = None
    occurred_at: datetime | None = None
    published_at: datetime | None = None
    captured_at: datetime
    source_ref: SourceRef
    # 无 logical_key / content_hash / observation_id / version / confidence / knowledge 字段
```

## 2. 交付 ② — Provider Adapter Contract（冻结）

### 2.1 职责边界（MUST / MUST NOT）

```
Provider Adapter（每源一个：iFinD / akshare / EastMoney / CNInfo / mock / crawler …）

✅ MUST（provider 负责）                        ❌ MUST NOT（provider 禁止）
────────────────────────────────────         ─────────────────────────────────────────
API 调用 / HTTP / SDK 取数                      persistence（写库；落 spine 是 ingestion 的事）
authentication（credential 从环境读）           hashing / identity generation（content_hash / logical_key）
rate limit / retry（源侧限流适配）              observation_id / version 赋值
source field mapping（源字段 → canonical）      调用 LLM / ReasoningPort（D6-3）
source semantic interpretation（懂自己字段名/   访问 Evolution（import market-intelligence / 读写 EvolutionStore）（D6-4）
  粒度/时间语义，映射到 predicate/fact_type）    产出 Knowledge（Belief/KnowledgeObject/Revision/View）（D6-2）
优雅降级（不支持的家族返回空 tuple 或            跨 provider 分支（共享层才禁 `if provider==`；provider 内只懂自己）
  NotImplementedError）
```

- 定性（ADR 0021 D6）：**Provider 是「事实的搬运工 + 翻译官」，不是「解释者」**。「翻译」= source field/semantic mapping（把源结构映射成 canonical 语义草稿）；「理解/判断/摘要」属 Evolution（M3.7），不属 provider。
- **语义映射必须在 provider 内**（D4a）：只有 iFinD adapter 懂 iFinD 字段名。上提到共享层就得写 `if provider == "ifind"`，即 ADR 与外部 Review **都反对**的耦合。
- **定身份/落库必须在共享层**（D4a）：`logical_key`/`content_hash`/幂等/provenance 挂载/写 spine 只在 ingestion 一处，换 provider 不触碰。

### 2.2 canonical 写侧契约（命名冻结）

canonical 契约名 **`ObservationProvider`**（解决与 legacy `MarketDataProvider` 撞名，ADR 0021 D3）：

```python
# contract sketch —— 非实现；M3.6.1 落码
@runtime_checkable
class ObservationProvider(Protocol):
    name: str
    def fetch_observations(self, query: DataQuery) -> tuple[ObservationDraft, ...]:
        ...
```

- 单方法 `fetch_observations → tuple[ObservationDraft, ...]`（统一 Observation，而非统一 Provider，D2）——**不采纳**多方法 `fetch_quote/fetch_financial/...` 富接口（那是往「统一 Provider」漂）。加 fact 家族只是多一个 `FactType`+`predicate`，契约不变。
- **新增 provider 一律实现 `ObservationProvider`**，不得以 legacy `PublicMarketDataProvider` 作为未来 provider 契约（D8 清单④）。

## 3. 交付 ③ — Commercial Provider Config Boundary（冻结）

### 3.1 核心原则：商业源是可配置 adapter，不是系统标准/依赖

```
免费环境（默认，无 token 可跑全链路）：  mock / akshare / EastMoney / CNInfo
增强环境（opt-in，需 credential）：      iFinD / Wind / Tushare
                    │
                    ▼
       无论哪个源，进入 ShanHai 的永远只有 ObservationDraft
```

- **iFinD / Wind 不能成为硬依赖**（D8 清单⑤）：免费额度断供即换源，`Knowledge / Context / Decision` **零改动**（D2/D5/D6-5）。
- 商业源仅经配置 opt-in 接入；**缺 credential 时不启用、不报错阻断**（优雅降级，D7）。

### 3.2 配置策略（冻结形态）

**环境变量（M3.6 落地形态，沿用 [factory.py](../../services/market-data/shanhai_market_data/factory.py) 的 `SHANHAI_MARKET_STORE` 风格）**：

```
# 免费模式（默认）
SHANHAI_DATA_PROVIDER=mock          # 或 =akshare

# 增强模式（opt-in）
SHANHAI_DATA_PROVIDER=ifind
IFIND_APP_KEY=...                   # 经环境注入；provider config 读取，绝不硬编码、绝不进 Observation/认知层
IFIND_SECRET=...
```

**声明式 provider 表（未来形态，本里程碑只登记方向、不实现，D7）**，兑现「商业源 `enabled: false` 默认关、免费源默认开」：

```yaml
# 未来形态（不实现）——商业源默认 disabled，credential 走 ${ENV} 占位
providers:
  ifind:   { enabled: false, api_key: ${IFIND_TOKEN} }
  wind:    { enabled: false }
  tushare: { enabled: true }        # commercial optional，已在 M3.2 保留
  akshare: { enabled: true }
  eastmoney: { enabled: true }
  cninfo:  { enabled: true }
# per-capability primary/fallback 路由亦属未来（不实现）：
#   market: { primary: ifind, fallback: [akshare] }
```

- provider 选择是**装配层关注点**，Context / Evolution 对其无感知（沿用 ADR 0019「禁 `if source == 'ifind'`」）。
- credential **只从环境读取**，源码中禁明文 `APP_KEY/SECRET`（D6-6）；credential 绝不进 `ObservationDraft` / 认知层。

## 4. 交付 ④ — Migration Plan（冻结）

### 4.1 legacy 路径 vs canonical 路径

```
旧（legacy，M3.2，保留不立即删）：
  PublicMarketDataProvider
        │  fetch_profile/security/quote/financial/announcement → *Record + SourceRef
        ▼
  mapper / fact_mapper
        │
        ▼
  MarketFact / FinancialFact  ──►  repository.upsert（latest 投影领域对象）

新（canonical，M3.6）：
  ObservationProvider（每源一 adapter）
        │  fetch_observations(DataQuery) → tuple[ObservationDraft, ...]
        ▼
  Observation Ingestion Pipeline（共享，唯一定身份处）
        │  logical_key / content_hash / 幂等 / provenance 挂载 / 写 spine
        ▼
  Observation Spine（append-only）  ──►  ObservationReadPort  ──►  Evolution / Context
```

### 4.2 迁移纪律（冻结，D3 R1）

1. **不立即删除旧路径**：M3.2 `PublicMarketDataProvider` + `acquisition.py` + mapper 链保留为已验证 legacy（跑通 EastMoney/CNInfo/Tushare + 11 套测试），强行推倒回归面过大。
2. **渐迁方向锁定**：各源 adapter **逐步迁移**到产出 `ObservationDraft` 走新 ingestion pipeline。`MarketFact`-投影路径与 `Observation`-append 路径并存**仅作迁移窗口，非终态**。
3. **避免两套 provider 词汇长期并行**：方向锁定「收敛到单一 `ObservationDraft` 输出契约」，**不允许无限期双轨**。legacy 退役**时点**另经 Review 定（不在 M3.6.0 拍死时点，只锁死**方向**）。
4. **新增 provider 不走 legacy**：任何新 provider 直接实现 `ObservationProvider` → `ObservationDraft`，不新增 `*Record` 家族契约。
5. **legacy 冻结能力（Review Gate 新增约束）**：legacy `PublicMarketDataProvider → Record → MarketFact` 路径**只维护、不新增能力**——不接新数据源、不加新 `*Record` 类型、不扩 `fetch_*` 方法。后续**所有新增数据源必须走 `ObservationDraft`**；否则 `Record → MarketFact` 会随新源持续膨胀，形成**永久双轨**（这正是纪律 3「不允许无限期双轨」的反面）。legacy 仅保留现有 EastMoney/CNInfo/Tushare 能力至退役。
