# ShanHai 产品愿景（Product Vision）

> 任何 AI Agent（Trae / Claude / GPT / Codex）或协作者读此文件即可理解 ShanHai 为什么存在、要解决什么问题、长期走向哪里。

## 一句话定位

ShanHai（山海）是一套 **AI Native 的中国资本市场认知与决策系统**。

它不是 AI 炒股机器人，也不是又一个行情/交易软件。它是一条认知流水线：

```
数据
 ↓
知识
 ↓
认知
 ↓
研究
 ↓
策略
 ↓
执行
```

## ShanHai 为什么存在

中国资本市场的信息是海量、分散、强时效、强关联的：财报、公告、新闻、政策、产业链、历史事件彼此交织。人脑无法长期、系统、无遗忘地消化这些信息，传统软件只做「展示」与「交易」，不沉淀「理解」。

ShanHai 的存在意义：把市场信息**持续编译为可被 AI 复用的知识与认知**，让分析与决策建立在长期积累的结构化知识之上，而不是一次性的对话或人工记忆。

## 要解决的核心问题

- 信息过载：数据多但难以形成连贯认知。
- 知识易失：研究结论散落在聊天记录/文档里，无法沉淀复用。
- 关系隐蔽：公司—行业—政策—产业链—事件之间的关系缺乏显式建模。
- 决策不可追溯：结论缺少从数据到判断的可回放链路。

## 与普通股票软件、量化系统的区别

| 维度 | 普通股票软件 | 量化系统 | ShanHai |
|---|---|---|---|
| 核心 | 行情展示 / 下单 | 因子 / 信号 / 回测 | 知识与认知沉淀 |
| 资产 | 实时行情 | 策略与因子库 | 知识库 + 产业链关系 + 研究经验 |
| AI 角色 | 几乎无 | 统计/机器学习模型 | AI Native：多 Agent 研究与推理 |
| 时间尺度 | 实时 | 高频 / 中低频 | 长期演进的认知系统 |
| 输出 | 价格 / 成交 | 交易信号 | 可追溯的研究与决策 |

ShanHai 不与量化系统对立：量化偏「数值信号」，ShanHai 偏「认知与理解」。执行层（含交易）是认知成熟后的下游产物，不是起点。

## AI 投资决策系统演进路线

```
Phase 0  Harness Foundation   —— 工程与运行框架（已完成）
Phase 1  Agent Runtime        —— Agent 执行模型 / 生命周期 / 运行记录（进行中）
Phase 2  Knowledge            —— Wiki Engine 信息提取 / 实体 / 关系 / 产业链
Phase 3  Research             —— 多 Agent 协作研究、证据链与评估
Phase 4  Strategy             —— 由认知驱动的策略生成
Phase 5  Execution            —— 交易执行（最后阶段，本阶段明确禁止）
```

## 本阶段不做什么

为保证认知基础扎实，以下方向当前**明确不实现**（详见 [PROJECT_STATE.md](PROJECT_STATE.md)）：

- 实时行情系统
- 券商交易接口
- 自动交易 Agent
- 复杂量化 / 高频策略
- 回测系统

> 核心信念：先有知识与认知能力，再谈策略与执行。ShanHai 优先把「理解市场」做扎实。
