**`KnowledgeReadPort` 的落点是本文第一个要定的实现细节**（见 §2）。其余子域（`knowledge/` 主体聚合物化、`reasoning/`、`memory/`、`evolution/`）本期**只登记不建**。

### 1.1 依赖边界如何被机器守住

延续 market-data 的 AST 校验模式（[test_dependency_boundary.py](../../tests/market_data/test_dependency_boundary.py)），本期新增两张校验网（实现期落地）：

## 2. `KnowledgeReadPort` 落点与签名（落地 D2 / O1）

ADR 0019 已定：新增独立只读 Protocol，签名固定：

```python
# contract sketch —— 非实现
@runtime_checkable
class KnowledgeReadPort(Protocol):
    def get_observations_as_of(
        self,
        subject: SubjectRef,
        *,
        knowledge_at: datetime,
        effective_at: datetime | None = None,
        fact_types: tuple[FactType, ...] = (),
    ) -> tuple[Observation, ...]:
        ...
```

- **语义**：返回该 subject、在 `captured_at ≤ knowledge_at` 约束下的所有 observation（append-only spine 的历史行，不是 latest 投影）。`effective_at` 若给出，进一步用 `occurred_at/published_at ≤ effective_at` 过滤（世界视角）。`fact_types` 为空 = 全部家族。
- **返回类型 `Observation`**：一个**新的只读 DTO**（§3），不是 `MarketFact`——因为 spine 行 = 「一条观测记录」，其身份是 `(logical_key, content_hash, captured_at)`，而 `MarketFact` 是 latest 投影后的领域对象。二者刻意分开，避免把「历史观测」和「当前信念」混成一个类型。

### 2.1 落点：两个候选，本文倾向 **A**

| 方案 | 描述 | 权衡 |
|--|--|--|
| **A（倾向）** | `KnowledgeReadPort` Protocol + 其 InMemory/SQLite 实现放在 **market-data 侧**（新目录 `ports/`）；market-intelligence 只 import 这个 Protocol | 读能力与被读的 spine 同包，SQLite 实现能直接复用 `sqlite_repository.py` 的连接/JSON 编解码；market-intelligence 保持「只依赖端口」纯净。**关键校验**：新代码必须仍满足 market-data 的 FORBIDDEN_IMPORTS（它本就只依赖 pydantic + stdlib sqlite3，天然满足） |
| B | Protocol 定义在 market-intelligence，实现（adapter）也在 market-intelligence，直接持有 SQLite 连接 | 违反「不泄漏存储细节到认知层」（ADR 0019 §5：market-intelligence 不 import 存储实现）。**不采纳** |

> 采用 A：`KnowledgeReadPort` 是 market-data 对外暴露的**第二个只读契约**（与 9 方法 `MarketKnowledgeRepository` 并列，互不修改）。market-intelligence 的 `ContextAssembler` 构造时注入一个 `KnowledgeReadPort`，永远不知道背后是 SQLite 还是 InMemory。

### 2.2 两个实现，parity 是正确性保证

延续 M3.3 Step 2「SQLite == InMemory read-model parity」策略：

- `InMemoryKnowledgeReadPort`：读 `InMemoryMarketKnowledgeRepository` 内部保留的 observation 列表（reference 语义）。
- `SQLiteKnowledgeReadPort`：`SELECT ... FROM knowledge_observation WHERE subject_id=? AND captured_at<=? [AND fact_type IN (...)] ORDER BY captured_at`，命中已有索引 `idx_observation_subject_type` 与 `idx_observation_logical_captured`（见 [sqlite_repository.py schema](../../services/market-data/shanhai_market_data/sqlite_repository.py#L152-L157)），**零 schema 变更**。
- parity 测试：同一批 observation 灌入两个后端，对任意 `(subject, knowledge_at, effective_at, fact_types)` 断言两端返回**逐字段相等**。

## 3. `Observation` 只读投影 DTO（§2 返回类型）

spine 行的只读投影，字段直接映射 `knowledge_observation` 列（见 [schema](../../services/market-data/shanhai_market_data/sqlite_repository.py#L133-L157)），**不新造存储字段**：

```python
# contract sketch —— frozen, extra="forbid"
class Observation(_FrozenModel):
    logical_key: str          # = 旧 fact_id（lineage）
    content_hash: str         # 幂等身份的一半
    fact_type: FactType
    subject: SubjectRef       # subject_type + subject_id (+ label from attributes)
    predicate: str | None
    object_value: str | None
    occurred_at: datetime | None    # effective / valid time
    published_at: datetime | None   # effective / valid time
    captured_at: datetime           # knowledge / transaction time（过滤主轴）
    confidence: float
    source_ref: SourceRef           # 复用现有 provenance，零新造
    # 结构化 detail 以引用/最小载荷携带，不在此展开成 typed 子表
```

- `Observation` 是**读侧 DTO**，不是新持久化实体。它 1:1 对应一条 spine 行；typed detail（financial/quote/announcement）在 Assembler 需要时再经现有投影路径取，避免把 detail 塞进这个通用 DTO。
- 不变量：`Observation` **不可变、不参与写**。Context 层永不修改 observation（ADR 0019 D4）。

## 4. `MarketContextSnapshot` 模型形状（落地 D3 / D4 / D7）

契约稿 §4.2 已给出 JSON 草图。这里定**Python 模型骨架**（frozen + extra=forbid，集合用 tuple，`schema_version` 用 `Literal`），字段命名复用既有领域词汇（`SubjectRef` / `SourceRef` / `FactType` / `TimeBasis`）：

```python
# contract sketch —— 非实现；所有 model frozen=True, extra="forbid"

class AsOf(_FrozenModel):
    effective_at: datetime      # 世界视角：截至这一刻成立/可见的事实
    knowledge_at: datetime      # 系统视角：只采纳 captured_at ≤ 此刻 的 observation

class Leaf(_FrozenModel):        # P4：每个数值都是三元组，不是裸值
    value: Any | None
    provenance: tuple[SourceRef, ...] = ()
    confidence: float = 1.0

class CognitionRef(_FrozenModel):   # historical_cognition：引用而非复制（禁 import experience）
    ref: str                    # experience/run/outcome/lesson 的 id
    at: datetime | None = None
    horizon: str | None = None

class DataQuality(_FrozenModel):    # D7：这份认知本身有多可信/完整/新鲜
    coverage: tuple[tuple[str, float], ...] = ()     # (section, 0..1)
    freshness_max_staleness_days: float | None = None
    stale_sections: tuple[str, ...] = ()
    conflicts: tuple[Conflict, ...] = ()             # 多源不一致，列 competing_sources
    trust_floor: SourceTrustLevel | None = None      # 本 snapshot 内最低 trust_level
    missing: tuple[str, ...] = ()                    # 显式缺失，不静默当 0

class MarketContextSnapshot(_FrozenModel):
    subject: SubjectRef
    as_of: AsOf
    identity: dict[str, Leaf]          # company_profile / industry / listing
    financial_state: dict[str, Leaf]   # 投影自 FinancialFact
    market_state: dict[str, Leaf]      # 投影自 QUOTE observation
    events: tuple[EventView, ...]      # 投影自 timeline，仅 as_of 之前
    historical_cognition: tuple[CognitionRef, ...] = ()
    data_quality: DataQuality
    snapshot_id: str                   # deterministic from (subject, as_of, source set)
    schema_version: Literal["market_context_snapshot.v1"]
    assembled_at: datetime
```

- **P4 结构化护栏**：每个数值经 `Leaf{value, provenance[], confidence}` 承载。`provenance[]` 元素即现有 `SourceRef`（provider/dataset/captured_at/raw_snapshot_ref/version/hash/trust_level），**零新造 provenance 词汇**。
- **命名护栏（D3）**：类型叫 `MarketContextSnapshot`，与 runtime-kernel 的 `RuntimeContext` 不撞名；**禁**裸 `Context` / `ContextBuilder`。
- **`snapshot_id` 确定性**：由 `(subject, as_of, 参与的 observation 身份集合)` 派生（如对 sorted `(logical_key, content_hash)` 集合 + as_of 做 hash）。这是「相同 `(subject, as_of)` → 相同 snapshot」（P2 可复现）的机器可验证锚点。

## 5. `ContextAssembler` 组装管线（落地 D4 / D5 / D6）

`ContextAssembler` 是本层唯一编排者，**只依赖一个 `KnowledgeReadPort`**（构造注入），不依赖 provider、SQLite、LLM。

```python
# contract sketch —— 非实现
class ContextAssembler:
    def __init__(self, read_port: KnowledgeReadPort, *, cognition_reader: CognitionRefReader | None = None):
        ...

    def assemble(self, subject: SubjectRef, as_of: AsOf) -> MarketContextSnapshot:
        # 1. 读 as_of 之前的 observation（bitemporal）
        obs = read_port.get_observations_as_of(
            subject, knowledge_at=as_of.knowledge_at, effective_at=as_of.effective_at,
        )
        # 2. 调和为 Knowledge：按 logical_key 分组，取 effective time ≤ effective_at 中最新一条
        knowledge = reconcile_latest_as_of(obs, as_of)
        # 3. 聚合为 subject 中心的分区（identity / financial_state / market_state / events）
        sections = project_sections(knowledge)
        # 4. 计算 data_quality（coverage/freshness/conflicts/trust_floor/missing）
        quality = compute_data_quality(obs, knowledge, as_of)
        # 5. 附 historical_cognition（只引用 id/ref，禁复制、禁 import experience）
        cognition = cognition_reader.refs_for(subject, as_of) if cognition_reader else ()
        # 6. 冻结为 MarketContextSnapshot（含 deterministic snapshot_id）
        return freeze_snapshot(subject, as_of, sections, quality, cognition)
```

关键点：

- **步骤 1「读」是唯一 I/O**；步骤 2–6 是**纯函数**（同一批 observation → 同一 snapshot），这是 P2 可复现 + D6 deterministic view 的实现根据。
- **组装期绝不调用 provider**（P3）：只读「已持久化的 observation」，不「现在去抓一遍」。
- **首版不落库（D6）**：`assemble` 返回内存对象，不写 `context_snapshot` 表。「快照可随时重算」由 append-only spine 保证。
- **调和规则（reconcile_latest_as_of）**：按 `logical_key` 分组 → 过滤 `effective time ≤ effective_at` → 取 `captured_at` 最新（当前信念）。冲突（同 logical_key 多源不一致）不静默取一个，而是进 `data_quality.conflicts`。
- `CognitionRefReader` 是一个**窄只读接口**（返回 `tuple[CognitionRef,...]`），本期可给 Noop 实现；它保证 historical_cognition 走「引用」而非 import experience 模块。

### 5.1 bitemporal 语义落地（D5）

- `knowledge_at = now` → 退化为「当前认知」（≈ current truth，但带 quality/provenance/historical_cognition）。
- `knowledge_at` = 过去某刻 → 「当时 AI 会看到的认知」（历史认知回放）。因 spine append-only + assemble 纯函数，回放**确定性**，无需保存当时快照副本。
- **首版只做 `knowledge_at` 过滤 + effective 排序**，不做完整 SQL:2011 区间 bitemporal。

## 6. 测试与 parity 策略

延续 market-data 的「stdlib 可跑、AST 守边界、双后端 parity」传统（见 [test_repository_behavior_suite.py](../../tests/market_data/test_repository_behavior_suite.py)）：

1. **`KnowledgeReadPort` parity**：同一批 observation 灌入 InMemory + SQLite 两个 `KnowledgeReadPort` 实现，对多组 `(subject, knowledge_at, effective_at, fact_types)` 断言返回逐字段相等（含 `captured_at` 边界、空 fact_types = 全部）。
2. **bitemporal 回放确定性**：先灌 T1 的 observation，再灌 T2 的更新（append，不覆盖）；断言 `as_of(knowledge_at=T1)` 的 snapshot **不含** T2 之后的 observation，且两次调用同参数得到相同 `snapshot_id`。
3. **Assembler 纯函数性**：固定一批 observation，`assemble(subject, as_of)` 调两次 → 深度相等；证明可复现（P2）。
4. **data_quality 正确性**：构造缺失/陈旧/冲突场景，断言 `missing[]` 显式列出、`stale_sections` 命中、`conflicts.competing_sources` 不静默、`trust_floor` = 最低 trust_level。
5. **依赖边界 AST 校验**：`market-intelligence` 不 import runtime-kernel、不 import experience（historical_cognition 只引用 ref）、无 trading 术语；`market-data` 现有禁令不回归。
6. **provider 无感知**：断言 market-intelligence 源码中不出现 `ifind`/`wind`/`akshare`/`SHANHAI_DATA_MODE` 等 provider 判定分支。

运行方式沿用：`PYTHONPATH=services/market-data:services/market-intelligence:. .venv/bin/python -m tests.market_intelligence.<suite>`。

## 7. 分步提交计划（每步到 Review Gate 停）

实现拆为最小可评审步骤，每步单独 commit、**不 push**、完成即停 Review Gate（延续项目 Review Gate 铁律）：

| Step | 范围 | 交付 | Gate |
|--|--|--|--|
| **S1** | `KnowledgeReadPort` Protocol + `Observation` DTO（market-data `ports/`），**不含实现** | 契约 + 类型 | 契约评审 |
| **S2** | `InMemoryKnowledgeReadPort` + `SQLiteKnowledgeReadPort`（复用现有 spine，零 schema 变更）+ parity 测试 | 双实现 + parity 绿 | 读能力评审 |
| **S3** | `services/market-intelligence` 包骨架 + `MarketContextSnapshot`/`AsOf`/`Leaf`/`DataQuality` 模型（frozen）+ AST 边界测试 | 模型 + 边界网 | 模型评审 |
| **S4** | `ContextAssembler.assemble` 组装管线（纯函数）+ reconcile/project/quality + 回放确定性测试 | 组装可跑 | 认知层评审 |

- **S1–S4 全程不接 provider、不接 LLM、不落 `context_snapshot` 表、不改 9 方法契约。**
- Reasoning Engine（M3.7）、Knowledge Object 物化（M3.5）、Data Provider premium 接入（M3.6）均**不在本里程碑**。
