# ADR 0007：Wiki 信息提取流程

状态：已采纳
日期：2026-06-22

## 背景

Phase 0 的 `wiki-engine` 只定义了知识 Schema（`Entity` / `Relation` / `Document`），`Document.entities` / `relations` 默认为空，没有任何把文档编译为结构化知识的逻辑。Phase 1 需要让「知识编译层」具备最小可用的信息提取能力：从文档内容中识别实体、发现关系，填充 `Document`。

这一步必须同时满足 AGENTS.md 的铁律：
- 模块独立：`wiki-engine` 不处理 Agent 逻辑。
- 模型解耦：业务代码不直接调用模型（§3.1）。
- 调用链：`Agent → Tool → Service → Database`（§4）。

难点在于「信息提取」天然想直接调用大模型，若让 `wiki-engine` 或 `Tool` 自己调模型，将违反铁律。

## 决定

1. **职责划分（三层）：**
   - **Service（`wiki-engine`）保持 model-agnostic。** 新增 `Extractor`，做**规则驱动**的确定性抽取：基于关键词词典识别实体，基于文本模式（如「X 属于 Y 行业」）发现关系，返回填充后的 `Document`。不依赖模型、不依赖网络，可单测、可复现。
   - **Tool（`WikiExtractTool`）** 包装 `Extractor`，提供统一的 `execute()` 接口，作为 Agent 触达 Service 的唯一通道。放在 `wiki-engine` 包内（依赖 `shanhai-tools` 的 `Tool` 基类），避免 `tools` 反向依赖 `wiki-engine`。
   - **Agent（`WikiExtractionAgent`）** 编排链路：`think` 阶段经 `ctx.complete(task=EXTRACTION)` 调 `ModelRouter`（证明模型在环、能力路由生效），`act` 阶段经 `ctx.use_tool("wiki_extract", ...)` 调 Tool。该 Agent 属**领域逻辑**，置于 `wiki-engine`（依赖 `agent-runtime`），保持 `agent-runtime` 作为通用基础设施不反向依赖任何领域模块。

2. **调用链（强制）：**
   ```
   WikiExtractionAgent
     ├─ think:  ctx.complete(prompt, task=EXTRACTION) → ModelRouter → Provider
     ├─ act:    ctx.use_tool("wiki_extract", document) → WikiExtractTool → Extractor(Service)
     └─ observe: done（单步）
   ```
   模型只出现在 Agent 的 `think`；结构化编译只出现在 Service；二者由 Tool 解耦。

3. **抽取策略（V0.1，规则驱动）：**
   - 实体：按 `EntityType` 维护关键词/别名词典，命中即产出 `Entity`。
   - 关系：按 `RelationType` 维护文本模式（如「属于…行业」→ `BELONGS_TO_INDUSTRY`），命中且两端实体均识别到即产出 `Relation`。
   - id 采用确定性生成（基于类型+名称），保证可复现与去重。

## 原因

- 规则驱动确定性抽取**现在就能跑通并单测**，不依赖真实 Provider（MockProvider 仅回显文本，无法解析 JSON），符合当前阶段「先把链路与边界做对」的目标。
- 把模型调用收口在 Agent、把编译收口在 Service，从结构上保证铁律不被绕过；未来切换为「模型驱动 JSON 抽取」时，只需替换 Agent 的 `think`/`act` 逻辑或新增 `Extractor` 实现，Service 与 Tool 边界不变。
- `Tool` 置于 `wiki-engine` 包内，保持依赖方向 `wiki-engine → tools`（单向），不污染通用 `tools` 包。

## 影响

- `services/wiki-engine` 新增 `extractor.py`（Service）、`tool.py`（`WikiExtractTool`）与 `agent.py`（`WikiExtractionAgent`），`pyproject.toml` 增加对 `shanhai-tools` 与 `shanhai-agent-runtime` 的依赖。
- `agent-runtime` 不感知 `wiki-engine`，依赖方向保持 `wiki-engine → {tools, agent-runtime}` 单向。
- 新增 `tests/test_wiki_extraction.py`：Extractor 单测 + Agent 链路集成测。
- 后续「模型驱动抽取」（接入真实 Provider 后）另行扩展，不改变本 ADR 的职责划分与调用链；提取结果落库留待数据层接入阶段，另开 ADR。
