## 0. 本设计的硬约束（不变量，继承 ADR 0020 §4/§5 + S4.0 §0 + R4）

- **doc-only**：本文不写任何 `.py`、不建 `services/market-intelligence/evolution/` 目录、不建测试文件。以下所有代码块均为**契约草图（contract sketch）**，非实现。
- **First scope 只做四件事**（§3）：`Domain model + NoopReasoner + Revision lifecycle + InMemory reference store`。**不做**：SQLite / LLM / iFinD / Wind / akshare / Tushare / ChangeDetector 触发策略 / confidence 聚合。
- **不接 LLM**：pipeline 的 reasoning 步只落 `ReasoningPort` 抽象（Protocol）+ deterministic `NoopReasoner` 参考实现；具体推理器 + LLM 属 reasoning-engine（M3.7）。
- **不改 schema、不落库**：KnowledgeObject 版本链首版**不落表**（对齐 ADR 0019 D6 + S4.0 §0），只在 **InMemory reference store** 表达；`knowledge_object` / `knowledge_revision` / `observation_history` 表一律不建（O5 留白）。
- **不推翻 0019**：deterministic Knowledge（D4）与 deterministic `ContextAssembler`（R1-4）保持冻结；本层是 additive 并行解释层（ADR 0020 D2）。
- **依赖方向冻结（沿用 ADR 0019 R1 + ADR 0020 §5 + D9）**：`runtime-kernel → reasoning-engine → market-intelligence → market-data`；`evolution/` 读 Observation 只经 `ObservationReadPort`（基元签名），不 import 存储实现；不 import reasoning-engine（只依赖 `ReasoningPort` Protocol）；**MUST NOT** import / 依赖 `MarketContextSnapshot`（D9，R4-1）。
- **RevisionGate 保持 deterministic（R4-2）**：只做机械可判定校验（`evidence exists / schema valid / version conflict / provenance complete`），禁 reasonableness/quality 判断。
- **Review Gate**：本文完成后不写实现。实现拆成小步 commit，每步仍到 Review Gate 停（§9）。

## 1. 交付 ①：Module boundary（落地 ADR 0020 §5 + S4.0 §7 + R4 指定布局）

### 1.0 四模块与依赖方向（沿用 ADR 0019 R1，永不反向）

```
services/
├── market-data          原始事实：Observation / spine / ObservationReadPort（M3.3+S3，冻结）
├── market-intelligence  认知世界：
│   ├── context/         Snapshot（M3.4 S1–S3，deterministic，R1-4，本轮不动）
│   └── evolution/       ← 本轮新增：Candidate / Revision / KnowledgeObject / Gate / NoopReasoner
├── reasoning-engine     推理世界：ReasoningPort 实现 + LLM（M3.7，本期只登记不建）
└── runtime-kernel       执行环境

依赖方向（import 箭头永远指向 market-data，绝不反向）：
    runtime-kernel ──► reasoning-engine ──► market-intelligence ──► market-data
                            │                      │
                            └─ 实现 ReasoningPort   └─ evolution/ 注入 ReasoningPort(Protocol)
```

**铁律（沿用 R1-1）**：`market-data` 永远不知道 `intelligence` 存在。本轮新增 `evolution/` 是 `market-intelligence` 内的子域，读 Observation 只经 `ObservationReadPort`（S3 已落地），**依赖方向仍单向指向 market-data**。

### 1.1 本轮落地目录（evolution 子域）

> **命名对齐 R4**：用户在 S4.0 Review 指定布局为 `market-intelligence/evolution/`（含 Candidate / Revision / KnowledgeObject）。S4.0 契约稿 §7 曾登记为 `knowledge/`，本文将其**收敛为 `evolution/`**（S4.0 §7 同步一行指向本文为 authoritative 布局）。

- **现有 `models.py` / `assembler.py` 不动**：本轮只新增 `evolution/` 子域，不改 M3.4 S1–S3 已冻结的 Context 侧代码。

## 2. First scope 的边界（落地 R4「建议只做」）

本轮 S4.1 **只做四件事**，其余全部留白：

| 做（S4.1 first scope） | 不做（明确留白，见 §10） |
|--|--|
| **Domain model**：第一批 6 个 frozen 模型（§4） | ❌ SQLite / 任何持久化（O5，仍 InMemory） |
| **NoopReasoner**：deterministic 参考推理器（§6） | ❌ LLM / ReasoningPort 具体实现（M3.7） |
| **Revision lifecycle**：版本链 + 状态机（§5） | ❌ ChangeDetector 触发策略（O2，仅留 Protocol 占位） |
| **InMemory reference store**：版本链内存表达（§8） | ❌ confidence 聚合算法（R4-3 / O4，保持开放） |
| | ❌ iFinD / Wind / akshare / Tushare（M3.6） |
| | ❌ 改 schema / 落 `knowledge_object` 表 |
| | ❌ 推翻 D4（deterministic Knowledge）/ R1-4（deterministic Assembler） |

**定性**：S4.1 是「用 deterministic NoopReasoner 把整条 pipeline 契约跑通、把版本链 append-only 语义在内存里坐实」的最小闭环。它证明**架构形状成立**（Observation → Candidate → reasoning → Gate → Revision → 版本链），但**不产生任何真实认知**（NoopReasoner 不推理，只做恒等映射）。真实认知等 M3.7 接入 ReasoningPort 后才有。

## 3. Pipeline 全景（落地 S4.0 §5，标注 S4.1 落地范围）

```
[1] Observation（append-only 事实，经 ObservationReadPort 只读；S3 已具备）
        │  ChangeDetector.detect(...)  ← S4.1 仅落 Protocol 占位，不实现触发策略（O2）
        ▼
[2] CandidateKnowledgeChange（结构化「信念修订候选」= 可验证假设，非 summary）  ★S4.1 冻结模型
        │  ReasoningPort.reason(candidate, evidence)  ← S4.1 = NoopReasoner（deterministic 恒等映射）
        ▼
[3] ProposedRevision（拟修订：拟 beliefs + delta + 各自 evidence_refs）
        │  RevisionGate.admit(current, proposed)  ← S4.1 = DeterministicRevisionGate（R4-2）
        ▼
[4] KnowledgeRevision → KnowledgeObject@vN+1（append，evidence-bound，禁覆盖 vN）  ★S4.1 冻结模型
        │  KnowledgeObjectStore.append(...)  ← S4.1 = InMemoryKnowledgeObjectStore
        ▼
    版本链 v1 → v2 → v3（旧版本永久保留，retire != delete）
```

- **S4.1 落地的步**：[2]（Candidate 模型）、[2]→[3]（NoopReasoner）、[3]→[4]（DeterministicRevisionGate）、[4]（Revision + 版本链 + InMemory store）。
- **S4.1 仅占位的步**：[1]→[2] 的 `ChangeDetector.detect`——只落 Protocol 签名，**不实现触发策略**（O2 留白）。S4.1 的测试**直接构造 `CandidateKnowledgeChange`** 喂入 pipeline，绕过检测阶段（等价 ADR 0017 Stage 2-b「Producer 只提出 Created 态候选，测试直接构造」）。
- **LLM 唯一入口 = [2]→[3] 且经 ReasoningPort**：禁 `Observation → LLM → Knowledge` 直连（ADR 0020 D6）。S4.1 用 NoopReasoner 占住这个位置，证明「pipeline 不接 LLM 也能跑通」。

## 4. 交付 ②：第一批冻结模型（落地 R4「建议冻结」的 6 个模型）

用户在 R4 明确要求 S4.1 冻结 6 个模型：`EvidenceRef / KnowledgeObject / Belief / CandidateKnowledgeChange / KnowledgeRevision / RevisionGateResult`。下面给出契约草图（对齐 S4.0 §2–§5 已定形状 + ADR 0017 Candidate 数据模型精神）。

> 全部 `frozen=True, extra="forbid"`；集合一律用 `tuple`；沿用 `evolution/models.py` 的 `_FrozenModel` 基类（与 Context 侧 `models.py` 同款 `ConfigDict`）。`SubjectRef` import 自 market-data（下游方向，允许）。

```python
# contract sketch —— services/market-intelligence/shanhai_market_intelligence/evolution/models.py
from __future__ import annotations
from datetime import datetime
from enum import Enum
from typing import Literal
from pydantic import BaseModel, ConfigDict
from shanhai_market_data.models import SubjectRef   # 下游 import，允许


class _FrozenModel(BaseModel):
    model_config = ConfigDict(frozen=True, extra="forbid")


# ── 枚举（受控词表，杜绝退化为自由文本 summary）──────────────────────
class EvidenceRelation(str, Enum):
    supports = "supports"
    contradicts = "contradicts"       # 反例也是证据
    contextualizes = "contextualizes"

class BeliefPolarity(str, Enum):
    supportive = "supportive"
    cautionary = "cautionary"
    neutral = "neutral"

class ChangeKind(str, Enum):
    new_evidence = "new_evidence"
    conflict = "conflict"
    gap = "gap"


# ── 模型 1：EvidenceRef —— 锚定一条 observation 的身份（不内嵌值，S4.0 §4）──
class EvidenceRef(_FrozenModel):
    logical_key: str            # observation 身份的一半（= 旧 fact_id / lineage）
    content_hash: str           # observation 身份的另一半（幂等锚点）
    captured_at: datetime       # 便于版本链复现 / 新鲜度排序，不是 value 本身
    relation: EvidenceRelation  # supports / contradicts / contextualizes


# ── 模型 2：Belief —— evidence-bound 的结构化信念（S4.0 §2）────────────
class BeliefStatement(_FrozenModel):
    dimension: str              # profitability / moat / valuation_regime …
    claim: str                  # 结构化 / 受控主张，非任意长文
    # summary: str | None       # 若保留仅人类可读派生视图，不作语义/匹配依据

class Belief(_FrozenModel):
    belief_id: str
    statement: BeliefStatement
    evidence_refs: tuple[EvidenceRef, ...]   # 必须非空（构造前置条件；Gate 强校验）
    confidence: float                        # 0~1，演化量（非静态 occurrence）
    polarity: BeliefPolarity
    # scope: BeliefScope | None = None       # 适用边界，S4.1 暂不建（O 留白）


# ── 模型 3：KnowledgeObject —— evidence-bound、versioned 的信念聚合（S4.0 §2）──
class KnowledgeObjectRef(_FrozenModel):      # 指向某个版本（版本链锚点）
    object_id: str
    version: int
    revision_id: str

class KnowledgeObject(_FrozenModel):
    subject: SubjectRef
    version: int                             # v1=1, v2=2 …
    previous_version: KnowledgeObjectRef | None = None   # v1 为 None（版本链锚点，禁覆盖）
    beliefs: tuple[Belief, ...] = ()
    evidence_refs: tuple[EvidenceRef, ...] = ()          # 各 belief evidence 的并集（引用）
    confidence: float = 0.0                  # 对象级综合置信度（派生自 beliefs；聚合算法留白 R4-3/O4）
    as_of_knowledge_at: datetime             # 本版本采纳的 observation 截止线（对齐 0019 D5）
    updated_at: datetime
    object_id: str                           # deterministic from subject —— 跨版本稳定
    revision_id: str                         # deterministic from (object_id, version, evidence 身份集)
    schema_version: Literal["s4.0"] = "s4.0"


# ── 模型 4：CandidateKnowledgeChange —— 结构化「信念修订候选」（S4.0 §5，镜像 ExperienceCandidate）──
class RevisionHypothesis(_FrozenModel):
    dimension: str              # 拟修订哪个信念维度
    claim: str                  # 结构化假设主张（非 summary）
    rationale: str              # 为何提出（人类可读，不作语义载体）

class CandidateRef(_FrozenModel):
    candidate_id: str
    hypothesis_version: int

class CandidateKnowledgeChange(_FrozenModel):
    candidate_id: str
    subject: SubjectRef
    change_kind: ChangeKind
    hypothesis: RevisionHypothesis
    source_refs: tuple[EvidenceRef, ...]     # 触发此候选的 observation（来源）
    evidence_refs: tuple[EvidenceRef, ...]   # 可用于验证的 observation（证据），与 source 分离
    hypothesis_version: int                  # 候选自身可演化（对齐 ADR 0017）
    created_at: datetime


# ── 模型 5：KnowledgeRevision —— 一次演化事件（版本链，禁覆盖，S4.0 §3）──
class BeliefDelta(_FrozenModel):
    added: tuple[str, ...] = ()              # 新增 belief_id
    revised: tuple[str, ...] = ()            # confidence/scope 变化的 belief_id
    retired: tuple[str, ...] = ()            # 不再持有的 belief_id（retire != delete：旧版本仍保留）

class KnowledgeRevision(_FrozenModel):
    revision_id: str                         # = 新版本的 revision_id
    object_id: str
    from_version: int | None                 # None = 初次创建（v1）
    to_version: int                          # from_version + 1（或 1）
    triggered_by: CandidateRef               # 哪个 Candidate 触发（来源）
    reasoning_ref: str | None = None         # ReasoningPort 产出的推理记录引用（M3.7；S4.1 = None）
    belief_delta: BeliefDelta
    created_at: datetime


# ── 模型 6：RevisionGateResult —— Gate 的 deterministic 判定结果（R4-2）──
class GateRejectReason(str, Enum):
    evidence_missing = "evidence_missing"        # 某 belief evidence_refs 为空
    evidence_unresolvable = "evidence_unresolvable"  # evidence 指向的 observation 不存在
    schema_invalid = "schema_invalid"            # proposed 结构不合契约
    version_conflict = "version_conflict"        # to_version 与 current 版本链冲突
    provenance_incomplete = "provenance_incomplete"  # provenance 缺失

class RevisionGateResult(_FrozenModel):
    admitted: bool
    reject_reasons: tuple[GateRejectReason, ...] = ()   # admitted=True 时为空
    revision: KnowledgeRevision | None = None           # admitted=True 时非 None
    knowledge_object: KnowledgeObject | None = None      # admitted=True 时 = 新版本 vN+1
    # 铁律（R4-2）：只携带 deterministic 校验结果 + 已固化的版本；不携带任何 LLM 自由文本
```

字段级铁律（本 contract 的护栏，继承 S4.0 §2/§4）：

- **belief 必须有 evidence**：`Belief.evidence_refs` 非空是构造前置条件，也是本层与 AI summary storage 的唯一硬边界。Gate 强校验（§7）。
- **结构化信念而非 summary**：`BeliefStatement.claim` / `RevisionHypothesis.claim` 是结构化主张；`summary` / `rationale` 若保留仅人类可读派生视图，不作语义/匹配依据。
- **`object_id` 跨版本稳定，`revision_id` 每版本唯一**：deterministic 派生（§5.3），保证版本链可确定性复现。
- **引用而非复制**：`EvidenceRef` 只持 observation 身份 + `captured_at`，不内嵌 value（避免事实与解释双写不一致）。
- **`RevisionGateResult` 是 Gate 唯一出口形状**：`admitted` + 拒因（deterministic 枚举）+ 已固化版本；**不携带** LLM 生成内容（R4-2）。

## 5. 交付 ③：Revision lifecycle（版本链 + 状态机，落地 S4.0 §3，镜像 ADR 0017）

### 5.1 版本链语义（append-only，禁覆盖）

```
object_id = KO(subject)
   │
   ├── v1 (revision_id=r1)  beliefs=[b1, b2]           previous_version = None
   │
   ├── v2 (revision_id=r2)  beliefs=[b1', b3]          previous_version → v1(r1)
   │                        delta: revised=[b1], added=[b3], retired=[b2]
   │
   └── v3 (revision_id=r3)  beliefs=[b1', b3, b4]      previous_version → v2(r2)
                            delta: added=[b4]
```

- **每次演化生成新版本（append）**，旧版本永久保留、不删除、不覆盖。
- **`retire != delete`**：v2 retire 掉 b2，只是「v2 不再包含 b2」；v1 里 b2 仍在——因此能回放「AI 在 v1 时为什么持有 b2」。
- **`belief_delta` 显式化**：不靠 diff 反推，Revision 自身记录 added/revised/retired，保证演化历史可审计。

### 5.2 Revision lifecycle 状态机（镜像 ADR 0017 Candidate lifecycle）

ADR 0017 的 Candidate 状态机是 `Created → Evaluating → Validated → Promoted`（+ Rejected/Archived/Reactivated 回退）。S4.1 的 Knowledge Evolution 是它在 market-knowledge 域的镜像，但**本轮 first scope 简化**：不实现 Validator/Evaluating 的多轮验证累积（那依赖 outcome 回填，属后续），只落 **Candidate 提出 → reasoning → Gate 判定 → 固化/拒绝** 的单遍生命周期。

```
        (测试直接构造 / 未来 ChangeDetector 产出)
                    │
                    ▼
              Proposed              CandidateKnowledgeChange 已提出，未推理
                    │  ReasoningPort.reason(candidate, evidence)
                    ▼
              Reasoned              ProposedRevision 已产出（NoopReasoner：恒等映射）
                    │  RevisionGate.admit(current, proposed)
              ┌─────┴─────┐
        admitted        rejected
              │             │
              ▼             ▼
          Committed      Rejected     （RevisionGateResult.admitted = True / False）
              │             │
              ▼             └──► 不产生新版本（版本链不变；拒因见 GateRejectReason）
     KnowledgeObject@vN+1
     append 到版本链（旧版本保留）
```

状态语义、触发者：

| 状态 | 含义 | 谁触发进入 | 进入条件 |
|------|------|-----------|----------|
| Proposed | 候选已提出，未推理 | **ChangeDetector（未来）/ 测试直接构造（S4.1）** | 结构化 `CandidateKnowledgeChange` 就位 |
| Reasoned | 拟修订已产出 | **ReasoningPort**（S4.1 = NoopReasoner） | `reason()` 返回 `ProposedRevision`，每条 belief evidence 非空 |
| Committed | 已固化为新版本 | **RevisionGate** | deterministic 校验全过（R4-2）；append vN+1 |
| Rejected | 候选被 Gate 拒绝 | **RevisionGate** | 任一 deterministic 校验失败（拒因见 §7.1） |

**与 ADR 0017 的差异（本轮简化，明确记录）**：

- **不落 Evaluating / Validated 的多轮验证累积**：ADR 0017 的 Validator 基于 outcome 回填做多次验证，S4.1 无 outcome 回填（属后续），故合并为单遍 `Proposed → Reasoned → Committed/Rejected`。
- **Rejected 不做 Archived/Reactivated 回退**：本轮 Rejected 即结束（不产生新版本，版本链不变），不实现「条件重现重新激活」。回退语义留待后续（对齐 ADR 0017 但暂不落地）。
- **不变量沿用**：状态只能由 ReasoningPort / RevisionGate 推进；测试/调用方**禁止**直接构造 `KnowledgeObject@vN+1` 绕过 Gate（RevisionGate 是唯一写入口，§7）。

### 5.3 deterministic id 派生（可复现锚点）

```
object_id   = deterministic_hash(subject 身份)                         # 跨版本稳定
revision_id = deterministic_hash(object_id, version, evidence 身份集)   # 每版本唯一
```

- `object_id` 只由 subject 决定，同一 subject 的所有版本共享同一 `object_id`（标识「关于该 subject 的认知这条演化线」）。
- `revision_id` 由 `(object_id, version, 该版本 evidence_refs 的排序身份集)` 决定——相同输入产出相同 `revision_id`，保证版本可确定性复现（对齐 Context 侧 `snapshot_id` deterministic 派生模式）。
- **实现细节留白**：具体 hash 算法（如 `sha256(canonical_json)`）在实现期定，本文只冻结「deterministic、跨版本稳定/每版本唯一」的语义契约。

## 6. 交付 ④：ReasoningPort + NoopReasoner（落地 S4.0 §5，对齐 ADR 0017 NoopValidator）

```python
# contract sketch —— services/market-intelligence/shanhai_market_intelligence/evolution/reasoning.py
from typing import Protocol

class ProposedRevision(_FrozenModel):
    candidate_id: str
    object_id: str
    proposed_beliefs: tuple[Belief, ...]     # 每条必须 evidence_refs 非空（否则被 Gate 拒）
    proposed_delta: BeliefDelta
    reasoning_ref: str | None = None         # M3.7 落地；S4.1 = None


class ReasoningPort(Protocol):               # 由 reasoning-engine（M3.7）实现；S4.1 用 NoopReasoner
    def reason(
        self,
        candidate: CandidateKnowledgeChange,
        evidence: tuple[EvidenceRef, ...],   # 候选携带的证据引用（值经 ObservationReadPort 按需取）
    ) -> ProposedRevision: ...
    # 铁律（ADR 0020 D3/D6）：reason 消费 候选+证据，产出「拟修订提案」；
    #   不写 KnowledgeObject、不生成 evidence；产出的每条拟 belief 必须携带 evidence_refs。


class NoopReasoner:                          # S4.1 deterministic 参考实现（对齐 ADR 0017 NoopValidator）
    """不推理、不总结：把 candidate.hypothesis 恒等映射为 proposed（evidence 直通）。

    用途：让整条 pipeline 契约在不接 LLM 的前提下可跑通、可测。它证明架构形状成立，
    但不产生任何真实认知——真实推理等 M3.7 接入 ReasoningPort。
    """
    def reason(self, candidate, evidence):
        # 1. 把 candidate.hypothesis 结构化映射为一条 Belief（claim/dimension 直通）
        # 2. belief.evidence_refs = candidate.evidence_refs（证据直通，保证非空 → 过 Gate）
        # 3. belief.confidence = 固定占位值（如 candidate 无 → 0.5；不做聚合，R4-3）
        # 4. proposed_delta = added=[新 belief_id]（首版）/ 依 candidate.change_kind 定 added/revised/retired
        # 5. reasoning_ref = None（S4.1 无推理记录）
        ...
```

铁律：

- **evolution 侧只 import `ReasoningPort` 抽象**，不 import reasoning-engine、不接任何 LLM（ADR 0020 D3；与 ADR 0011「Agent 禁直调模型」同源）。
- **NoopReasoner deterministic**：相同 candidate → 相同 ProposedRevision，保证 pipeline 可确定性测试。
- **NoopReasoner 不生成 evidence**：它只把 candidate 已有的 `evidence_refs` 直通，不凭空造证据（守 evidence-first）。若 candidate 无 evidence，产出的 belief evidence 为空 → 被 Gate 拒（§7），这正是 evidence-first 的护栏被测试覆盖的点。

## 7. 交付 ⑤：RevisionGate（deterministic 唯一写入口，落地 R4-2）

### 7.1 Gate 契约（只做机械可判定校验，禁 reasonableness/quality）

```python
# contract sketch —— services/market-intelligence/shanhai_market_intelligence/evolution/gate.py
from typing import Protocol

class EvidenceResolver(Protocol):            # 校验 evidence 是否指向真实 observation（经 ObservationReadPort）
    def exists(self, ref: EvidenceRef) -> bool: ...

class RevisionGate(Protocol):
    def admit(
        self,
        current: KnowledgeObject | None,     # 当前最新版本（v1 时为 None）
        proposed: ProposedRevision,
    ) -> RevisionGateResult: ...


class DeterministicRevisionGate:             # S4.1 参考实现（R4-2）
    """KnowledgeObject 唯一写入口。只做 deterministic 校验，只固化不创作。

    允许的校验（机械可判定，R4-2）：
      - evidence exists?        每条 proposed_belief.evidence_refs 非空
      - evidence resolvable?    每条 evidence 指向真实 observation（经 EvidenceResolver）
      - schema valid?           proposed 结构合契约（模型层已 extra=forbid，Gate 再查 delta 一致性）
      - version conflict?       to_version == (current.version + 1)（或 current is None 时 == 1）
      - provenance complete?    evidence_refs 的 logical_key/content_hash 非空

    禁止的判断（R4-2，属 Reasoning Engine / M3.7）：
      - is this belief reasonable?   ← 需理解，禁
      - is this company good?        ← 需判断，禁
      - 任何对信念「质量/合理性」的评估

    定性：Reasoning 创造候选，RevisionGate 验证候选。
    """
    def __init__(self, resolver: EvidenceResolver) -> None: ...
    def admit(self, current, proposed):
        # 1. 逐条校验（deterministic）；任一失败 → RevisionGateResult(admitted=False, reject_reasons=[...])
        # 2. 全过 → 构造 KnowledgeRevision + KnowledgeObject@vN+1（append，evidence-bound）
        #    - version = (current.version + 1) if current else 1
        #    - previous_version = KnowledgeObjectRef(current) if current else None
        #    - revision_id = deterministic_hash(object_id, version, evidence 身份集)（§5.3）
        #    - 只把已校验的 proposed 固化为 vN+1，不新增/改写信念内容
        # 3. 返回 RevisionGateResult(admitted=True, revision=..., knowledge_object=vN+1)
        ...
```

### 7.2 Gate 铁律（继承 ADR 0017 Promotion Gate / ADR 0018 ArtifactBuilder「只固化不创作」）

- **唯一写入口**：任何 `KnowledgeObject` 版本的产生都必须经 `RevisionGate.admit`；**禁**任何路径直接构造 `KnowledgeObject@vN+1` 绕过 Gate。
- **只固化不创作**：Gate 只把已校验的 `proposed` 固化为新版本，**不新增/改写信念内容**，**禁**携带 LLM 自由文本作为信念内容载体。
- **deterministic（R4-2）**：Gate 的判定只依赖输入（current + proposed + observation 存在性），相同输入 → 相同 `RevisionGateResult`；**不含**任何需理解/判断的评估。
- **evidence-first 强校验**：每条 `proposed_belief.evidence_refs` 非空且指向真实 observation，否则拒（拒因 `evidence_missing` / `evidence_unresolvable`）。

## 8. 交付 ⑥：InMemory reference store（版本链内存表达，不落库 O5）

```python
# contract sketch —— services/market-intelligence/shanhai_market_intelligence/evolution/store.py
from typing import Protocol

class KnowledgeObjectStore(Protocol):
    """KnowledgeObject 版本链存储抽象。首版仅 InMemory；SQLite 属后续（O5，需 parity 套件）。"""
    def append(self, revision: KnowledgeRevision, knowledge_object: KnowledgeObject) -> None: ...
    def latest(self, object_id: str) -> KnowledgeObject | None: ...
    def get_version(self, object_id: str, version: int) -> KnowledgeObject | None: ...
    def version_chain(self, object_id: str) -> tuple[KnowledgeObject, ...]: ...   # v1..vN 有序
    def as_of(self, object_id: str, knowledge_at: datetime) -> KnowledgeObject | None: ...
        # 选 as_of_knowledge_at <= knowledge_at 的最大版本（bitemporal 复现，对齐 0019 D5）


class InMemoryKnowledgeObjectStore:          # S4.1 参考实现
    """进程内版本链存储：object_id → 有序 (version → KnowledgeObject) + revision 记录。

    append-only 语义（禁覆盖 / 禁删除）在此坐实：
      - append 只允许 to_version == latest.version + 1（或首版 == 1），否则 raise（version_conflict）
      - 不提供 delete / update 方法（结构上杜绝覆盖）
    """
    def __init__(self) -> None:
        self._chains: dict[str, list[KnowledgeObject]] = {}      # object_id → [v1, v2, ...]
        self._revisions: dict[str, list[KnowledgeRevision]] = {}
```

- **首版不落库（O5）**：版本链只在内存表达，对齐 ADR 0019 D6「deterministic view，落表是优化非正确性需求」。
- **append-only 结构性保证**：`InMemoryKnowledgeObjectStore` **不提供** `delete` / `update` 方法——从 API 表面就杜绝覆盖（比运行时校验更强）。
- **未来 parity 策略（登记，不实现）**：SQLite 版本链存储属后续（O5），届时**复用 S3 已验证的 InMemory↔SQLite parity 套件模式**（见 [test_observation_read_port_parity.py](../../tests/market_data/test_observation_read_port_parity.py)）——同一组 append/查询序列在两个 backend 上产出相同版本链，AST + behavior 双守护。本轮**只做 InMemory**，parity 套件不建。
