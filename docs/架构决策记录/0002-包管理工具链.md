# ADR 0002：Monorepo 包管理工具链

状态：已采纳
日期：2026-06-22

## 背景

Monorepo 同时包含 Python 服务与 Node 前端，需要确定多包管理工具，保证模块边界清晰、依赖可隔离、安装快。

## 决定

- Python 侧使用 **uv** 管理工作区（workspace），根 `pyproject.toml` 声明 workspace 成员，各 `services/*` 与 `packages/*` 各自带 `pyproject.toml`。
- Node 侧使用 **pnpm** workspace，根 `pnpm-workspace.yaml` 声明 `apps/*`。

## 原因

- uv 安装快、原生支持 workspace、可统一锁定依赖。
- pnpm 磁盘高效、workspace 成熟。
- 两者均为各自生态的现代主流方案，符合"长期演进"目标。

## 影响

- 开发者需安装 uv 与 pnpm（README 提供安装指引）。
- 若本地缺失 uv，仍可用标准 `python -m venv` + `pip install -e` 退化运行（Phase 0 已保证各包可被 editable 安装）。
- 锁文件 `uv.lock` / `pnpm-lock.yaml` 纳入版本控制。
