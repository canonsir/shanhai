## 0. 本设计的硬约束（不变量，继承 ADR 0021 §4/§5）

- **doc-only**：本文不写任何 `.py`，不建 provider 目录/测试文件。
- **不接数据源**：iFinD/Wind/akshare/Tushare 一律不接；不申请 iFinD APP_KEY/SECRET。
- **不改 schema**：不改 `knowledge_observation` spine / typed detail；沿用既有 `record()` / `record_many()` 写侧 API。
- **不动 legacy**：不改 legacy `MarketDataProvider`/`FinancialDataProvider`/`AnnouncementDataProvider`（`sync.py` 用）；不推倒 M3.2 `PublicMarketDataProvider` + `acquisition.py` + mapper 链（标 legacy 渐迁）。
- **统一 Observation 而非统一 Provider**：系统内部只认 `Observation`；provider 只产 `ObservationDraft`，身份由共享 pipeline 定。
- **Data Provider Boundary（6 条，ADR 0021 D6）**：provider 只产 Observation / 不产 Knowledge / 不调 LLM / 不访问 Evolution / 可替换 / credential 配置化。
- **依赖方向冻结**：Data Provider 在 `market-data` 内，永不 import `market-intelligence`。
- **Review Gate**：本文完成后不写实现。M3.6.0（doc-only Contract Review）→ M3.6.5 逐段经 Review。

## 1. 落点与目录布局

Data Provider Layer 落 `services/market-data`（Observation Store 所有者），与既有 M3.2 acquisition 同域。目标目录（本文只登记，不建）：

```
services/market-data/shanhai_market_data/
├── provider.py                    # 既有：PublicMarketDataProvider(M3.2) + legacy MarketDataProvider（保留不动）
├── acquisition.py                 # 既有：PublicDataAcquisitionService（legacy 渐迁路径，保留不动）
├── ports/
│   └── observation_reader.py      # 既有：Observation（读侧投影）+ ObservationReadPort（保留不动）
│
├── observation_provider.py        # 新增(M3.6.1)：ObservationProvider Protocol + DataQuery + ObservationDraft
├── ingestion.py                   # 新增(M3.6.3)：ObservationIngestionPipeline（定身份 + 幂等写 spine）
└── providers/                     # 既有目录（M3.2 EastMoney/CNInfo 在此），新增 canonical adapter 子包
    ├── __init__.py                #   既有：CninfoAnnouncementProvider / EastMoneyProvider（保留）
    ├── mock/                      #   新增(M3.6.2)：确定性 mock ObservationProvider
    │   └── client.py
    ├── ifind/                     #   新增(M3.6.4)：优先真实 provider（可替换）
    │   ├── client.py              #     取数（SDK/HTTP），不算身份、不写库
    │   ├── mapper.py              #     iFinD 结构 → ObservationDraft
    │   └── config.py              #     credential 装配（读环境变量，不硬编码）
    └── akshare/                   #   新增(M3.6.5)：免费默认 provider
        ├── client.py
        └── mapper.py
```

- **每个 provider = 一个 adapter 子包**：`client`（取数）+ `mapper`（源结构 → `ObservationDraft`）+ 可选 `config`（credential）。
- iFinD/akshare/mock 平级，同实现 `ObservationProvider`；换源不影响 `ingestion.py` 与下游。
- **legacy 与 canonical 并存（仅迁移窗口）**：M3.2 的 `acquisition.py`（产 `MarketFact`）继续跑；canonical 走 `ObservationDraft → ingestion → Observation`。二者并存**仅作为迁移窗口，不作为终态**——方向锁定「收敛到单一 `ObservationDraft` 输出契约」，不允许两套 provider 词汇无限期双轨（ADR 0021 D3 R1）。退役时点另经 Review 定。
- **Migration strategy（M3.6.0 冻结项之一）**：`PublicMarketDataProvider` 不立即废弃；先经 M3.6.0 Contract Review 冻结边界，其后 EastMoney/CNInfo adapter 逐步迁移到 `ObservationDraft` 输出，`MarketFact`-投影路径待收敛后退役。

## 2. 交付 ①：Provider Contract（M3.6.0 Review 冻结 → M3.6.1 落码）

### 2.1 `DataQuery`（provider 输入）

```python
# contract sketch —— 非实现；frozen=True, extra="forbid"，集合用 tuple
# 属 market-data；只用 market-data 自己的类型（SubjectRef / FactType），不 import intelligence

class DataQuery(_FrozenModel):
    """向一个 provider 请求某 subject 的某些 fact 家族的观测。

    provider 据此取数并映射成 ObservationDraft。query 只表达「要什么」，
    不表达「怎么算身份」「落哪张表」——那是 ingestion pipeline 的事。
    """
    subject: SubjectRef                    # 复用 market-data 既有 SubjectRef（entity_type/entity_id/label）
    fact_types: tuple[FactType, ...] = ()  # 为空 = provider 尽力返回其支持的全部家族
    since: datetime | None = None          # 可选：只要 captured/occurred 在此之后的（provider 侧尽力）
    until: datetime | None = None
    limit: int | None = None               # 可选：单次上限（provider 侧尽力）
```

### 2.2 `ObservationDraft`（provider 输出，写侧草稿，**无身份**）

```python
# contract sketch —— 非实现
class ObservationDraft(_FrozenModel):
    """provider 产出的「语义观测草稿」——有语义、无身份。

    刻意区别于 ports/observation_reader.py 的 Observation（读侧投影，带
    logical_key/content_hash 身份）：draft 是 provider 给出的原始语义观测，
    logical_key / content_hash 由共享 ingestion pipeline 确定性算出（§3），
    provider 绝不自算身份。这是「统一 Observation 而非统一 Provider」的落点。
    """
    subject: SubjectRef                    # provider 侧已解析到 subject（未解析则经 ingestion 前的 resolver）
    fact_type: FactType                    # quote / financial / announcement / news / industry ...
    predicate: str | None = None          # 细分语义键，如 "valuation.pe_ttm"（进入 logical_key 计算）
    object_value: str | None = None       # 观测值（字符串承载，跨源统一；结构化 detail 走 typed 表）
    occurred_at: datetime | None = None    # effective time（事件发生）
    published_at: datetime | None = None   # effective time（市场可见）
    captured_at: datetime                  # knowledge time（provider 取到此观测的时刻）
    source_ref: SourceRef                  # provenance：provider 身份只活在这里（provider / trust_level / hash ...）
```

- **draft 与 `Observation` 字段几乎同构，唯独没有 `logical_key` / `content_hash`**——这正是 §2.1（ADR 0021）张力的解法：provider 不背身份职责。
- `object_value` 用字符串承载跨源统一（沿用现有 `Observation.object_value: str | None`）；iFinD 富字段/结构化明细经 typed detail 表投影，不塞 draft 顶层（ADR 0021 D5）。

### 2.3 `ObservationProvider`（canonical 写侧契约，解决 `MarketDataProvider` 撞名）

```python
# contract sketch —— 非实现
@runtime_checkable
class ObservationProvider(Protocol):
    """canonical 写侧 provider 契约（M3.6）。所有真实/mock 源实现它。

    Boundary（ADR 0021 D6）：只产 ObservationDraft；不产 Knowledge；不调 LLM；
    不访问 Evolution；可替换；credential 配置化。一个 provider 对不支持的
    fact 家族可返回空 tuple 或 raise NotImplementedError（沿用 M3.2 优雅降级）。
    """
    name: str

    def fetch_observations(self, query: DataQuery) -> tuple[ObservationDraft, ...]:
        ...
```

- 命名裁决：用户 spec 的 `MarketDataProvider` 与 legacy 撞名（[provider.py](../../services/market-data/shanhai_market_data/provider.py#L71)），canonical 改名 **`ObservationProvider`**（ADR 0021 D3）。
- 返回 `tuple[..., ...]`（frozen 精神），非 `list`。

## 3. 交付 ②：Observation Ingestion Pipeline（M3.6.3，共享，唯一定身份处）

```python
# contract sketch —— 非实现
class ObservationIngestionPipeline:
    """provider 无关的归一化管线：ObservationDraft → 带身份 Observation → 写 spine。

    唯一一处算身份/幂等/落库。换 provider 不触碰这里——这是「统一 Observation」的落点。
    依赖注入：一个 provider（ObservationProvider）+ 一个 observation writer（既有
    record()/record_many() 的持有者）。pipeline 不 import intelligence。
    """
    def __init__(self, provider: ObservationProvider, *, writer: _ObservationWriter) -> None:
        ...

    def ingest(self, query: DataQuery) -> IngestReport:
        drafts = self._provider.fetch_observations(query)     # provider 只搬运 + 翻译
        observations = tuple(self._materialize(d) for d in drafts)  # 定身份
        self._writer.record_many(observations)                # 幂等写 append-only spine
        return IngestReport(provider=self._provider.name, ingested=len(observations), ...)

    @staticmethod
    def _materialize(draft: ObservationDraft) -> Observation:
        logical_key  = _logical_key(draft.subject, draft.fact_type, draft.predicate)  # 身份稳定键
        content_hash = _content_hash(draft)                                            # 语义值哈希
        return Observation(logical_key=logical_key, content_hash=content_hash, ...draft 字段...)
```

管线三步（ADR 0021 D4）：

1. **归一化/校验草稿**：`fact_type` 合法、`subject` 已解析、`captured_at` 必填、时间戳一致。
2. **定身份（确定性）**：
   - `logical_key` = 稳定标识（`subject.entity_type:entity_id` + `fact_type` + `predicate`），标识「同一条逻辑事实的时间线」——同一 PE_TTM 时间线共享一个 `logical_key`。
   - `content_hash` = 对语义值（`object_value` + 关键时间戳 + `predicate` 等）确定性哈希——值变则 hash 变，产生新的一行；值不变则同 hash，幂等去重。
   - **具体哈希算法/键格式在 M3.6.3 实现期定**，本文只冻结「在 pipeline 一处、确定性、幂等 on `(logical_key, content_hash)`」原则。
3. **幂等写 spine**：走既有 `record_many()`（两个 adapter 已实现，`(logical_key, content_hash)` 去重、append-only、不覆盖历史）。

- **写侧完成后，下游经既有 `ObservationReadPort.query(subject, knowledge_at=...)` 读回**——Evolution 消费的仍是它已认识的 `Observation`，M3.6 只是给它喂真实/mock 数据，Evolution/Context **零改动**。
- `_ObservationWriter` 是对既有 `record()` / `record_many()` 的窄接口封装（M3.6.3 定；避免 pipeline 直接依赖具体 adapter）。

## 5. 交付 ④：映射示例（源结构 → ObservationDraft）

**iFinD（PE_TTM）**：

```
iFinD 返回：  最新市盈率  PE_TTM = 15.6  @ 2026-07-01（600519）
        │  ifind/mapper.py（机械映射，无推理、无 LLM）
        ▼
ObservationDraft(
    subject      = SubjectRef(entity_type="security", entity_id="600519", label="贵州茅台"),
    fact_type    = FactType.FINANCIAL,
    predicate    = "valuation.pe_ttm",
    object_value = "15.6",
    captured_at  = 2026-07-01T...,
    source_ref   = SourceRef(source_id="ifind", source_name="同花顺 iFinD",
                             provider="ifind", trust_level=SourceTrustLevel.OFFICIAL, hash=...),
)
        │  ingestion._materialize（定身份）
        ▼
Observation(logical_key="security:600519|financial|valuation.pe_ttm",
            content_hash="<hash(15.6 + captured_at + predicate)>", ...同上字段...)
```

**mock（确定性）**：同一 `DataQuery` 恒定产出同一批 draft → 同一批 `Observation`（同 `logical_key`/`content_hash`）→ 幂等；用于链路可复现验证。

**akshare**：字段名不同（如 `pe_ttm` 列），经 `akshare/mapper.py` 映射到**同一** `predicate="valuation.pe_ttm"` → 同一 `logical_key`——**换源不改下游身份**，这就是「统一 Observation」的收益。

- 每段独立 Review Gate 后开工，不一次性实现。

## 7. Boundary 守护（实现期落地，本文登记）

沿用既有依赖边界 AST 校验（ADR 0019 R1「market-data 禁 import intelligence」）扩展 6 条 Boundary（ADR 0021 D6）：

```
在 providers/** 与 ingestion.py 上：
- 禁 import shanhai_market_intelligence.*（Boundary 4：不访问 Evolution）
- 禁 import 任何 LLM/model-router/reasoning 符号（Boundary 3：不调 LLM）
- 禁产出 Belief / KnowledgeObject / KnowledgeRevision / KnowledgeView（Boundary 2：不产 Knowledge）
- provider 不出现 logical_key/content_hash 赋值（Boundary：不自算身份，归一化只在 ingestion）
- credential 只从环境读取，源码中禁明文 APP_KEY/SECRET（Boundary 6：配置化）
```

具体守护用例在 M3.6.1/M3.6.3 随代码落地（对齐 Case 18 AST 守护风格）。
