## 1. 模块布局（落地 D1 / O3 + R1-1）

新增独立包 `services/market-intelligence`，与既有 `market-data` / `runtime-kernel` 平级，包名 `shanhai_market_intelligence`（延续 `shanhai_*` 约定，见 [experience-evolution 布局](../../services/experience-evolution/shanhai_experience_evolution/)）。

### 1.0 未来四模块结构与依赖方向（R1-1）

ShanHai 认知栈最终收敛为四层，各层职责与依赖方向如下（`import` 箭头永远指向下游 market-data，绝不反向）：

```
services/
├── market-data          原始事实世界：Quote / FinancialFact / AnnouncementFact / Observation / Repository
├── market-intelligence  认知世界：KnowledgeObject / ContextAssembler / MarketContextSnapshot / Cognition
├── reasoning-engine     推理世界：Agent / Decision / Simulation（M3.7，本期只登记不建）
└── runtime-kernel       执行环境

依赖方向（R1 修订）：
    runtime-kernel ──► reasoning-engine ──► market-intelligence ──► market-data
    （新增 reasoning-engine 层，插在 runtime-kernel 与 market-intelligence 之间）
```

**铁律（R1-1）**：**`market-data` 永远不知道 `intelligence` 存在**。禁止 `market-data import KnowledgeObject`（或任何 intelligence 概念：`MarketContextSnapshot` / `AsOf` / `KnowledgeObject`）。一旦反向 import，market-data 的领域模型会被认知概念污染，重蹈 everything-data。**读方向**是 `market-intelligence ──(ObservationReadPort 只读)──► market-data`；**依赖方向**（谁 import 谁）永远指向 market-data。

M3.4 只落地链条最右两段（`market-intelligence → market-data`）；`reasoning-engine` 本期只在此登记，不建目录、不写代码。

### 1.1 本期落地目录


### 1.2 依赖边界如何被机器守住

延续 market-data 的 AST 校验模式（[test_dependency_boundary.py](../../tests/market_data/test_dependency_boundary.py)），本期新增两张校验网（S1 起落地）：

- `market-intelligence` 的 AST 校验：**禁 import** `shanhai_runtime_kernel`（不得反向依赖 runtime），**禁 trading 术语**（broker/place_order/position/…），`cognition_state` 侧**禁 import** `shanhai_experience*`（只引用 id/ref）。

## 2. `ObservationReadPort` 落点与签名（落地 D2 / O1 + R1-2）

**R1-2 命名裁决**：原文 `KnowledgeReadPort` 改名 **`ObservationReadPort`**（落 `services/market-data/shanhai_market_data/ports/observation_reader.py`）。理由：market-data 当前职责是 **Observation Store**，不是 Knowledge Store；`Knowledge*` 命名会暗示「market-data 拥有 knowledge」，污染领域所有权。真正的 `Observation → Knowledge Extraction → KnowledgeObject` 属 market-intelligence 内部；届时真正的 `KnowledgeReadPort` 应属 intelligence。

新增独立只读 Protocol，签名固定：

```python
# contract sketch —— 非实现；放 services/market-data/.../ports/observation_reader.py
@runtime_checkable
class ObservationReadPort(Protocol):
    def query(
        self,
        subject: SubjectRef,
        *,
        knowledge_at: datetime,
        effective_at: datetime | None = None,
        fact_types: tuple[FactType, ...] = (),
    ) -> tuple[Observation, ...]:
        ...
```

- **顶层入口 `query(subject, as_of)`**：概念上就是「给一个 subject 和一个 as_of，返回那一刻可见的 observation」。落到签名时，`as_of` 被**摊平为基元参数** `knowledge_at` / `effective_at`——**不能**用 market-intelligence 的 `AsOf` 类型（R1-1 铁律：market-data 永不 import intelligence 概念）。intelligence 侧的 `ContextAssembler` 从自己的 `AsOf` 拆出两个 datetime 再调 `query`。
- **语义**：返回该 subject、在 `captured_at ≤ knowledge_at` 约束下的所有 observation（append-only spine 的历史行，不是 latest 投影）。`effective_at` 若给出，进一步用 `occurred_at/published_at ≤ effective_at` 过滤（世界视角）。`fact_types` 为空 = 全部家族。
- **返回类型 `Observation`**：一个**新的只读 DTO**（§3），**属于 market-data**（与端口同包），不是 `MarketFact`——因为 spine 行 = 「一条观测记录」，其身份是 `(logical_key, content_hash, captured_at)`，而 `MarketFact` 是 latest 投影后的领域对象。二者刻意分开，避免把「历史观测」和「当前信念」混成一个类型。

### 2.1 落点：两个候选，本文采用 **A**

| 方案 | 描述 | 权衡 |
|--|--|--|
| **A（采用）** | `ObservationReadPort` Protocol + `Observation` DTO + 其 InMemory/SQLite 实现放在 **market-data 侧**（新目录 `ports/`）；market-intelligence 只 import 这个 Protocol | 读能力与被读的 spine 同包，SQLite 实现能直接复用 `sqlite_repository.py` 的连接/JSON 编解码；market-intelligence 保持「只依赖端口」纯净。**关键校验**：新代码必须仍满足 market-data 的 FORBIDDEN_IMPORTS（它本就只依赖 pydantic + stdlib sqlite3，天然满足），且**不 import 任何 intelligence 概念**（R1-1） |
| B | Protocol 定义在 market-intelligence，实现（adapter）也在 market-intelligence，直接持有 SQLite 连接 | 违反「不泄漏存储细节到认知层」（ADR 0019 §5：market-intelligence 不 import 存储实现）。**不采纳** |

> 采用 A：`ObservationReadPort` 是 market-data 对外暴露的**第二个只读契约**（与 9 方法 `MarketKnowledgeRepository` 并列，互不修改）。market-intelligence 的 `ContextAssembler` 构造时注入一个 `ObservationReadPort`，永远不知道背后是 SQLite 还是 InMemory。
>
> **S1 边界**：本步只落 Protocol + `Observation` DTO（interface only，**不含任何实现**）。InMemory 实现在 S2、SQLite 实现在 S3。

### 2.2 两个实现，parity 是正确性保证（S2 + S3）

延续 M3.3 Step 2「SQLite == InMemory read-model parity」策略：

- `InMemoryObservationReadPort`（S2，已落地）：**自持 append-only** observation 列表的 reference 实现（`record`/`record_many` 幂等于 `(logical_key, content_hash)`，`query` 做纯确定性过滤）。刻意**不复用** `InMemoryMarketKnowledgeRepository`——那是 current-truth、按 `fact_id` 覆盖、不保留历史（见 [test_repository_behavior_suite.py](../../tests/market_data/test_repository_behavior_suite.py#L300-L372)），无法回放「截至某 knowledge_at 看见过哪些观测」。先验证领域，不接 SQLite。`effective_at` **接受但不解释**（S2-3，保留给 S4）。
- `SQLiteObservationReadPort`（S3）：`SELECT ... FROM knowledge_observation WHERE subject_id=? AND captured_at<=? [AND fact_type IN (...)] ORDER BY captured_at`，命中已有索引 `idx_observation_subject_type` 与 `idx_observation_logical_captured`（见 [sqlite_repository.py schema](../../services/market-data/shanhai_market_data/sqlite_repository.py#L152-L157)），**零 schema 变更**。
- parity 测试（S3）：同一批 observation 灌入两个后端，对任意 `(subject, knowledge_at, effective_at, fact_types)` 断言两端返回**逐字段相等**。

## 3. `Observation` 只读投影 DTO（§2 返回类型，属 market-data）

`Observation` 是 **market-data 自己的只读 DTO**（与 `ObservationReadPort` 同包，落 `ports/observation_reader.py`），是 spine 行的只读投影，字段直接映射 `knowledge_observation` 列（见 [schema](../../services/market-data/shanhai_market_data/sqlite_repository.py#L133-L157)），**不新造存储字段**。它是 market-data 对外暴露的读契约类型；market-intelligence import 它是「下游 import 上游」，符合 R1-1 依赖方向（intelligence → market-data）。

```python
# contract sketch —— frozen, extra="forbid"; 属 market-data，非 intelligence
class Observation(_FrozenModel):
    logical_key: str          # = 旧 fact_id（lineage）
    content_hash: str         # 幂等身份的一半
    fact_type: FactType
    subject: SubjectRef       # subject_type + subject_id (+ label from attributes)
    predicate: str | None
    object_value: str | None
    occurred_at: datetime | None    # effective / valid time
    published_at: datetime | None   # effective / valid time
    captured_at: datetime           # knowledge / transaction time（过滤主轴）
    confidence: float
    source_ref: SourceRef           # 复用现有 provenance，零新造
    # 结构化 detail 以引用/最小载荷携带，不在此展开成 typed 子表
```

- `Observation` 是**读侧 DTO**，不是新持久化实体。它 1:1 对应一条 spine 行；typed detail（financial/quote/announcement）在 Assembler 需要时再经现有投影路径取，避免把 detail 塞进这个通用 DTO。
- 不变量：`Observation` **不可变、不参与写**。Context 层永不修改 observation（ADR 0019 D4）。

## 4. `MarketContextSnapshot` 模型形状（落地 D3 / D4 / D7 + R1-3 ref-based）

**R1-3 裁决**：保持命名 `MarketContextSnapshot`，但**禁止**它成为数据聚合容器 / 万能对象。第一版形状严格收紧为 **ref-based**，只有 7 个认知内容字段：

```
{ subject, as_of, observation_refs, knowledge_refs, market_state, cognition_state, data_quality }
```

**废弃旧草图**：契约稿 §4.2 与本文旧版 §4 里按数据种类平铺的 `identity / financial_state / events / historical_cognition` 分区**作废**。原因（R1-3）：`{financials:{}, news:{}, technical:{}, chip:{}, ...}` 这种「今天有哪些数据」的平铺形状，半年后必然长成 daily_stock_analysis 的 `ContextPack`。

**核心区别**：Snapshot = 「某个时间点，系统**认为自己知道什么**」，**不是**「今天有哪些数据」。因此它不内嵌数据值，而是**引用**（ref）已持久化的 observation/knowledge，外加系统在该时刻的**状态**（market_state / cognition_state）与**自我质量评估**（data_quality）。

```python
# contract sketch —— 非实现；所有 model frozen=True, extra="forbid"；集合用 tuple

class AsOf(_FrozenModel):
    effective_at: datetime      # 世界视角：截至这一刻成立/可见的事实
    knowledge_at: datetime      # 系统视角：只采纳 captured_at ≤ 此刻 的 observation

class ObservationRef(_FrozenModel):   # 引用一条 observation（身份，不内嵌值）
    logical_key: str
    content_hash: str
    fact_type: FactType               # import 自 market-data（下游方向，允许）
    captured_at: datetime             # 便于排序/新鲜度，不是值本身

class KnowledgeRef(_FrozenModel):     # 引用「某 logical_key 调和后的当前信念」（仍是 ref，非值）
    logical_key: str
    resolved_from: ObservationRef     # 该信念来自哪条 observation（可复现锚点）
    confidence: float = 1.0

class MarketState(_FrozenModel):      # 系统当下对该 subject 的「认知覆盖状态」，非行情数据
    known_fact_families: tuple[FactType, ...] = ()   # 当前有知识的家族（deterministic 投影）
    trust_floor: SourceTrustLevel | None = None      # 本 snapshot 内最低 trust_level
    # v1 只放确定性状态标记，不放判断/结论（ContextAssembler 第一版 deterministic）

class CognitionRef(_FrozenModel):     # cognition_state 成员：引用而非复制（禁 import experience）
    ref: str                          # experience/run/outcome/lesson 的 id
    kind: str                         # previous_analysis / previous_prediction / feedback
    at: datetime | None = None
    horizon: str | None = None

class CognitionState(_FrozenModel):   # 「我以前怎么想 / 预测 / 被验证」——全 ref
    refs: tuple[CognitionRef, ...] = ()

class Conflict(_FrozenModel):
    logical_key: str
    competing_sources: tuple[SourceRef, ...] = ()    # 多源不一致，不静默取一个

class DataQuality(_FrozenModel):      # D7：这份认知本身有多可信/完整/新鲜
    coverage: tuple[tuple[str, float], ...] = ()     # (dimension, 0..1)
    freshness_max_staleness_days: float | None = None
    stale_dimensions: tuple[str, ...] = ()
    conflicts: tuple[Conflict, ...] = ()
    trust_floor: SourceTrustLevel | None = None
    missing: tuple[str, ...] = ()                    # 显式缺失，不静默当 0

class MarketContextSnapshot(_FrozenModel):
    # ── 7 个认知内容字段（R1-3 严格 shape）────────────────────────────
    subject: SubjectRef
    as_of: AsOf
    observation_refs: tuple[ObservationRef, ...] = ()   # 该 as_of 参与的 observation（引用）
    knowledge_refs: tuple[KnowledgeRef, ...] = ()       # 调和后的当前信念（引用）
    market_state: MarketState
    cognition_state: CognitionState
    data_quality: DataQuality
    # ── 可复现 meta（非认知内容；P2 确定性锚点，测试用）────────────────
    snapshot_id: str                                    # deterministic from (subject, as_of, observation 身份集合)
    schema_version: Literal["market_context_snapshot.v1"]
    assembled_at: datetime
```

- **ref-based 护栏（R1-3）**：内容只有 `observation_refs` / `knowledge_refs`（引用已持久化事实/信念，不内嵌值）+ `market_state`（确定性覆盖状态，非行情数据）+ `cognition_state`（历史认知引用）+ `data_quality`（自我质量评估）。**不出现** `financials/news/technical/chip` 这类按数据种类平铺的内嵌容器。
- **provenance 保留（D7）**：值不进 snapshot，但 provenance/quality 依然完整——`ObservationRef` 携带身份与 `captured_at`，冲突/信任度进 `data_quality`（`SourceRef` 零新造）。需要具体值时，消费方持 ref 回 market-data 取（保持 snapshot 轻量、可复现）。
- **命名护栏（D3）**：类型叫 `MarketContextSnapshot`，与 runtime-kernel 的 `RuntimeContext` 不撞名；**禁**裸 `Context` / `ContextBuilder`。
- **`snapshot_id` 确定性**：由 `(subject, as_of, 参与的 observation 身份集合)` 派生（对 sorted `(logical_key, content_hash)` 集合 + as_of 做 hash）。这是「相同 `(subject, as_of)` → 相同 snapshot」（P2 可复现）的机器可验证锚点。`snapshot_id/schema_version/assembled_at` 是 meta，不是认知内容，不破坏 7 字段严格 shape。

## 5. `ContextAssembler` 组装管线（落地 D4 / D5 / D6 + R1-4 deterministic）

**R1-4 裁决**：`ContextAssembler` 第一版**纯 deterministic，不智能化**。它**只负责**：查询 / 过滤 / 排序 / as_of 计算 / provenance 合并 / quality 计算。**禁止**：推理 / 总结 / 判断 / LLM 自动总结。任何「结论/观点/评级/摘要」都不在 M3.4——那属 reasoning-engine（M3.7）。这样边界才不混乱：Context 层产出「系统知道什么」的确定性快照，推理层才产出「据此该怎么想」。

`ContextAssembler` 是本层唯一编排者，**只依赖一个 `ObservationReadPort`**（构造注入），不依赖 provider、SQLite、LLM。

```python
# contract sketch —— 非实现；S1 只落骨架（方法体 NotImplementedError），管线 S4 实现
class ContextAssembler:
    def __init__(self, read_port: ObservationReadPort, *, cognition_reader: CognitionRefReader | None = None):
        ...

    def assemble(self, subject: SubjectRef, as_of: AsOf) -> MarketContextSnapshot:
        # 1. 读 as_of 之前的 observation（bitemporal）——唯一 I/O
        #    AsOf 在此摊平为基元，端口不认识 AsOf（R1-1）
        obs = read_port.query(
            subject, knowledge_at=as_of.knowledge_at, effective_at=as_of.effective_at,
        )
        # 2. 调和为 Knowledge：按 logical_key 分组，取 effective time ≤ effective_at 中最新一条
        knowledge = reconcile_latest_as_of(obs, as_of)          # deterministic
        # 3. 投影为 ref：observation_refs / knowledge_refs / market_state（确定性覆盖状态）
        observation_refs = to_observation_refs(obs)             # deterministic
        knowledge_refs = to_knowledge_refs(knowledge)           # deterministic
        market_state = project_market_state(knowledge)          # 仅确定性状态标记，无判断
        # 4. 计算 data_quality（coverage/freshness/conflicts/trust_floor/missing）
        quality = compute_data_quality(obs, knowledge, as_of)   # deterministic
        # 5. 附 cognition_state（只引用 id/ref，禁复制、禁 import experience）
        cognition = CognitionState(refs=cognition_reader.refs_for(subject, as_of)) \
                    if cognition_reader else CognitionState()
        # 6. 冻结为 MarketContextSnapshot（含 deterministic snapshot_id）——无推理/总结
        return freeze_snapshot(subject, as_of, observation_refs, knowledge_refs,
                               market_state, cognition, quality)
```

关键点：

- **步骤 1「读」是唯一 I/O**；步骤 2–6 是**纯函数**（同一批 observation → 同一 snapshot），这是 P2 可复现 + D6 deterministic view 的实现根据。
- **纯 deterministic（R1-4）**：无 LLM、无网络、无随机、无「总结/判断」。`market_state` 只放确定性覆盖标记（如 `known_fact_families`），不放结论。
- **组装期绝不调用 provider**（P3）：只读「已持久化的 observation」，不「现在去抓一遍」。
- **首版不落库（D6）**：`assemble` 返回内存对象，不写 `context_snapshot` 表。「快照可随时重算」由 append-only spine 保证。
- **调和规则（reconcile_latest_as_of）**：按 `logical_key` 分组 → 过滤 `effective time ≤ effective_at` → 取 `captured_at` 最新（当前信念）。冲突（同 logical_key 多源不一致）不静默取一个，而是进 `data_quality.conflicts`。
- `CognitionRefReader` 是一个**窄只读接口**（返回 `tuple[CognitionRef,...]`），本期可给 Noop 实现；它保证 cognition_state 走「引用」而非 import experience 模块。

### 5.1 bitemporal 语义落地（D5）

- `knowledge_at = now` → 退化为「当前认知」（≈ current truth，但带 quality/provenance/cognition_state）。
- `knowledge_at` = 过去某刻 → 「当时 AI 会看到的认知」（历史认知回放）。因 spine append-only + assemble 纯函数，回放**确定性**，无需保存当时快照副本。
- **首版只做 `knowledge_at` 过滤 + effective 排序**，不做完整 SQL:2011 区间 bitemporal。

## 6. 测试与 parity 策略

延续 market-data 的「stdlib 可跑、AST 守边界、双后端 parity」传统（见 [test_repository_behavior_suite.py](../../tests/market_data/test_repository_behavior_suite.py)）：

1. **`ObservationReadPort` parity（S3）**：同一批 observation 灌入 InMemory + SQLite 两个 `ObservationReadPort` 实现，对多组 `(subject, knowledge_at, effective_at, fact_types)` 断言返回逐字段相等（含 `captured_at` 边界、空 fact_types = 全部）。
2. **bitemporal 回放确定性（S4）**：先灌 T1 的 observation，再灌 T2 的更新（append，不覆盖）；断言 `as_of(knowledge_at=T1)` 的 snapshot **不含** T2 之后的 observation，且两次调用同参数得到相同 `snapshot_id`。
3. **Assembler 纯函数性（S4）**：固定一批 observation，`assemble(subject, as_of)` 调两次 → 深度相等（same observation + same as_of = same snapshot）；证明可复现（P2）。
4. **data_quality 正确性（S4）**：构造缺失/陈旧/冲突场景，断言 `missing[]` 显式列出、`stale_dimensions` 命中、`conflicts.competing_sources` 不静默、`trust_floor` = 最低 trust_level。
5. **依赖边界 AST 校验（S1 起）**：`market-intelligence` 不 import runtime-kernel、不 import experience（cognition_state 只引用 ref）、无 trading 术语；`market-data` 现有禁令不回归，且**不 import 任何 intelligence 概念**（R1-1）。
6. **provider 无感知（S1 起）**：断言 market-intelligence 源码中不出现 `ifind`/`wind`/`akshare`/`SHANHAI_DATA_MODE` 等 provider 判定分支。
7. **ref-based shape 守护（S1）**：断言 `MarketContextSnapshot` 字段恰为 7 认知字段 + 3 meta，`extra="forbid"`；无 `financials/news/technical/chip` 类平铺容器（R1-3）。

运行方式沿用：`PYTHONPATH=services/market-data:services/market-intelligence:. .venv/bin/python -m tests.market_intelligence.<suite>`。

## 7. 分步提交计划（每步到 Review Gate 停）

实现拆为最小可评审步骤，每步单独 commit、**不 push**、完成即停 Review Gate（延续项目 Review Gate 铁律）。**R1-5 重排**：先立领域（skeleton）→ 再定读契约（InMemory）→ 再落持久化（SQLite parity）→ 最后组装认知（assembler）。目的是**先验证领域语言与依赖方向，再接数据库**，避免过早被存储细节牵着走。

| Step | 范围 | 交付 | Gate |
|--|--|--|--|
| **S1 — Domain skeleton** | `services/market-intelligence` 包骨架（`__init__.py` re-export）+ 领域模型（`MarketContextSnapshot` ref-based / `AsOf` / `ObservationRef` / `KnowledgeRef` / `MarketState` / `CognitionRef` / `CognitionState` / `Conflict` / `DataQuality`，frozen + extra=forbid）+ `ObservationReadPort` Protocol 与 `Observation` 只读 DTO（market-data `ports/`，**interface only 无实现**）+ `ContextAssembler` 骨架（`NotImplementedError`）+ AST 依赖边界测试 + ref-based shape 守护测试。**不接任何数据库**。 | 领域语言 + 依赖方向 + 边界网 | 领域/契约评审 |
| **S2 — Read contract** | `InMemoryObservationReadPort`：读 `InMemoryMarketKnowledgeRepository` 保留的 observation 列表（reference 语义），实现 `query(...)` 的过滤/排序/`fact_types`。**不接 SQLite**——先验证领域读契约。 | 读契约可跑（InMemory） | 读能力评审 |
| **S3 — Persistence adapter** | `SQLiteObservationReadPort`（复用现有 spine + 索引，**零 schema 变更**）+ parity 测试：同一批 observation 灌入两个后端，对任意 `(subject, knowledge_at, effective_at, fact_types)` 断言 `InMemory == SQLite` 逐字段相等。 | SQLite 实现 + parity 绿 | 持久化/parity 评审 |
| **S4 — Context assembly** | `ContextAssembler.assemble(subject, as_of)` 实现纯函数管线 + reconcile/project/quality；断言 `same observation + same as_of = same snapshot`（回放确定性、纯函数性）。 | 组装可跑 + 可复现 | 认知层评审 |

- **`ObservationReadPort` Protocol + `Observation` DTO 与全部领域模型都在 S1 落地（interface only）；实现分两步：InMemory 在 S2、SQLite 在 S3。`MarketContextSnapshot` 模型也在 S1 落地，其组装管线在 S4 实现。**
- **S1–S4 全程不接 provider、不接 LLM、不落 `context_snapshot` 表、不改 9 方法契约。**
- Reasoning Engine（M3.7）、Knowledge Object 物化（M3.5）、Data Provider premium 接入（M3.6）均**不在本里程碑**。
