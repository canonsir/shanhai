## 0. 本文定位（M3.6.1 前置 Gate 是什么 / 不是什么）

M3.6.0 冻结了「单个 provider 的输出契约」（`ObservationDraft`）与「共享层做什么」。M3.6.1 开工前，还差一件事没冻结：**多个 provider 在系统里如何被摆放、组合、选择**。本文只解决这一件事。

```
是：  冻结 provider 分类 / 默认组合 / optional 插槽 / credential 边界 / selection 边界 —— 装配边界（composition root）
不是：写 adapter / 写 registry 代码 / 写 selection policy 代码 / 接任何源 / 申请 token / 改 ObservationDraft 契约
```

**本文不新增 provider 输出契约**（那已在 M3.6.0 冻结）——只冻结「装配这一层」的边界，让 M3.6 后续接 iFinD 时，系统不会围绕 iFinD 设计，而是「先把数据源当插件，再决定是否购买增强能力」。

## 1. D1 — Provider 分类（冻结）

不按「免费 / 收费」二分，因为「不一定收费」≠「开箱即用」。按**开箱即用能力**分层。

### 1.1 「免费默认」的收紧定义（冻结）

一个 provider 属于 **Default** 当且仅当**全部**满足：

| 条件 | 必须 |
|--|--|
| 无 token | ✅ |
| 无付费账户 | ✅ |
| 默认启用（无需 opt-in） | ✅ |
| 可作为系统基础数据入口 | ✅ |

任何一条不满足 → 不是 Default。

### 1.2 四层 provider（冻结）

```
ObservationProvider（M3.6.0 冻结的单一契约，四层都实现它）
        │
        ├── Default Providers            开箱即用、无 credential、默认启用、系统基础数据入口（canonical 默认）
        │       ├── EastMoneyProvider    （已存在，M3.2；免费/无 token）
        │       └── CninfoAnnouncementProvider（已存在，M3.2；免费公开披露）
        │
        ├── Optional Free Providers      免费但非 canonical：免费聚合 adapter，opt-in、无 credential
        │       └── akshare              （未接；免费聚合 SDK，**永远不是 canonical 数据源**）
        │
        ├── Optional Commercial Providers  能力增强、需 credential/账户、默认关闭、opt-in
        │       ├── Tushare              （已存在 tushare.py；需 token + 积分 + 权限等级）
        │       ├── iFinD                （未接；商业授权）
        │       └── Wind                 （未接；商业授权）
        │
        └── Test / Reference Provider     确定性 mock，仅测试/链路复现，非生产数据源
                └── mock                 （M3.6.2 才写）
```

- **四层实现同一个 `ObservationProvider` 契约**（M3.6.0 §2.2 冻结的单方法 `fetch_observations(DataQuery) -> tuple[ObservationDraft, ...]`）——分类只影响**装配与选择**，不影响**输出契约**。换类别不改契约。
- **Default vs Optional Free 的分界**：两者都免费、无 token，但 **Default 是 canonical 默认数据入口**（ShanHai 直连的源），**Optional Free 是免费聚合 adapter**（如 akshare，套在别的源之上的聚合 SDK，opt-in、永不 canonical）。akshare 免费但**不满足「canonical 基础入口」**，故不进 Default（见 D2）。
- **Tushare 明确归 Optional Commercial**（本次用户裁决）。理由：虽多数基础接口免费，但架构上**需 token / 有积分体系 / 有权限等级 / 数据能力依赖账户状态**——不满足 D1「无 token」「无付费账户」，故与 iFinD/Wind 同层，**不进免费默认组合**。
- **mock 单列为 Test/Reference**：它不是数据源，是链路确定性复现工具（M3.6.0 §3.1 已把 mock 与免费源并列启用，但语义上它属测试基座，不是「基础数据入口」）。单列避免它被误当作 Default 数据源统计。

### 1.3 分类归属原则（冻结）——ADR 记准则，Composition Root 记实现

- **ADR 只记录「默认 provider 必须满足什么准则」，不记录「谁是默认实现」。** 例如 ADR 只写「默认 provider 必须：免费 / 无 token / 可直接启用 / 覆盖基础行情」，**不写**「默认 = EastMoney」。
- **「真正是谁」（EastMoney / CNInfo）写在本文（Composition Root Design）或 Application Config。** 这样以后默认源换（EastMoney → Sina → Tencent）**ADR 根本不用动**，只改组合根/配置。
- 本原则为 ShanHai 后续所有 ADR 的固定纪律（本次用户裁决），M3.6.1 是其首个落点。

## 2. D2 — 默认组合（冻结）

```
Default Providers（开箱即用，无 credential，默认启用）
    ├── EastMoneyProvider              profile / security / quote / financial（push2 / F10）
    └── CninfoAnnouncementProvider     disclosed announcement（巨潮资讯）
```

- 默认组合是**能力互补**而非冗余：EastMoney 覆盖行情/财务/画像，CNInfo 覆盖披露公告（EastMoney 无干净披露公告 feed，见 [eastmoney.py](../../services/market-data/shanhai_market_data/providers/eastmoney.py#L134-L135)）。两者合起来构成 ShanHai 无 token 可跑的**基础数据入口**。
- 默认组合内的 provider **平级**，均实现 `ObservationProvider`；谁供哪个 `FactType` 由 provider 自己的能力声明决定，不由业务层硬编码。
- **akshare 不在默认组合，归 Optional Free**（见 §1.2、§6 Reconcile R1）：akshare 是**免费采集聚合 SDK**（PROJECT_STATE 已定「AkShare 不作为核心依赖，它是采集聚合 SDK，不是数据源」），ShanHai 默认走**直连免费源**（EastMoney/CNInfo）而非再套一层聚合 SDK。**akshare 永远不是 canonical**——它只是一个可 opt-in 的免费聚合 adapter，不属于本文冻结的默认组合。

## 3. D3 — Optional 插槽（冻结）

```
Optional Commercial Providers（默认关闭，需 credential，opt-in 增强）
    ├── Tushare      需 token + 积分 + 权限等级
    ├── iFinD        商业授权（短期可用；能力必须保留配置，未来可能自费）
    └── Wind         商业授权
```

- 三者是**同层的可配置插槽**，没有谁被特殊处理。iFinD 只是 `CommercialObservationProvider` 之一——**系统不围绕 iFinD 设计**，iFinD 接入不改变默认组合、不改变下游认知层。
- 未来商业 provider 统一入口（**方向登记，本文不建目录、不写代码**）：

```
providers/
    commercial/
        tushare.py
        ifind.py
        wind.py
        …
```

- **Optional provider 缺 credential 时不启用、不报错阻断**（优雅降级，ADR 0021 D7）；免费额度断供即换源，`Knowledge / Context / Decision` 零改动（ADR 0021 D2/D5/D6-5）。
- 兑现用户要求「iFinD 只能短期使用，但收费源能力必须保留配置，未来可能自费」——iFinD 作为 Optional 插槽**永久保留配置位**，启停只是 `enabled` 开关，不是删代码/加代码。

### 3.1 配置模型（冻结形态）

**不采用**扁平配置（会让所有 provider 看起来同等级）：

```yaml
# ❌ 不采用：扁平——Default 与 Commercial 混为一谈
providers:
  tushare: { enabled: true }
```

**采用**按能力分层的声明式配置（**方向冻结，本里程碑不实现**）：

```yaml
# ✅ 冻结形态：defaults 是系统基础能力，optional 是能力增强
providers:

  defaults:                 # 系统基础能力：默认启用，无 credential
    - eastmoney
    - cninfo

  optional:                 # 能力增强：默认关闭，opt-in + credential
    tushare:
      enabled: false        # commercial optional（需 token/积分/权限）
    ifind:
      enabled: false        # 商业授权
    wind:
      enabled: false        # 商业授权
```

- 语义分层：`defaults` = 系统基础能力（列表，默认全开）；`optional` = 能力增强（映射，默认全关，逐个 opt-in）。
- credential 走 `${ENV}` 占位注入，**不写进 yaml 明文**（见 §4）。
- 这只是**装配层配置**，Context / Evolution 对其无感知（§5）。

## 4. D4 — Credential Boundary（冻结）

**provider 自己声明它需要什么 credential，但不能读取业务配置。**

```
✅ MUST（provider 负责）                      ❌ MUST NOT（provider 禁止）
──────────────────────────────────         ──────────────────────────────────────────
声明自身所需 credential（如 IFIND_TOKEN）     读取业务/认知层配置（Knowledge/Context/Evolution 的开关或参数）
从注入的 config/env 读取自己的 credential      硬编码 APP_KEY/SECRET（ADR 0021 D6-6）
缺 credential 时优雅降级（不启用、不阻断）      把 credential 写进 ObservationDraft / SourceRef / 任何 Observation 字段
                                            决定「用不用自己」（那是组合根/selection 的事，见 D5）
```

- credential **只从环境/注入的 config 读取**，源码禁明文（延续 ADR 0021 D6-6、M3.6.0 §3.2）。
- provider 声明「我需要 `IFIND_TOKEN`」是**能力声明**；provider 读「Context 该不该启用某认知」是**越界**——后者根本不是 provider 的关注点。
- credential **绝不进** `ObservationDraft` / `SourceRef` / 认知层（延续 M3.6.0 §1.3 禁止字段精神：provider 只吐语义草稿，不吐配置/密钥）。

## 5. D5 — Selection Boundary（冻结）

**组合根负责选择用哪个 provider；选择逻辑不下沉到任何层。**

### 5.1 Provider Priority 不属于 Provider 自身

**禁止**在 provider 内部或业务层写这种优先级/回退逻辑：

```python
# ❌ 禁止：provider 优先级污染业务/数据入口层
if ifind_available:
    use_ifind()
else:
    eastmoney()
```

否则 iFinD 一接入就污染业务层，且每加一个源都要改这些分支。

### 5.2 选择只在组合根

```
Application Composition Root      ← 唯一知道「有哪些 provider、谁优先、谁回退」的地方
        │
        ▼
Provider Selection Policy         ← 组合根内的策略（primary/fallback/per-capability）
        │
        ▼
ObservationProvider（被选中的那个/那批）
        │
        ▼
Observation Ingestion Pipeline    ← 只认 ObservationProvider 契约，不认「是哪个源」
        │
        ▼
Observation Spine → Evolution → Context   ← 对 provider 选择完全无感知
```

- **Selection 不进** `ingestion` / `observation` / `knowledge` / `context` —— 这四层永远只见 `ObservationProvider` 契约与 `Observation`，永不见「是 iFinD 还是 EastMoney」（延续 ADR 0019「禁 `if source == 'ifind'`」、ADR 0021 D6-5「provider 可替换」）。
- `ObservationProviderRegistry`（装配层组件，**M3.6.1 才写**）持有三类 provider；`Selection Policy`（primary/fallback/per-capability 路由）是组合根内的关注点，**本文只冻结边界，不实现策略**（多 provider 路由仍是 ADR 0021 D7 登记的未来项）。
- 收益：换源 / 加源 / 买增强能力，改动**只发生在组合根**，认知六层链路一行不动。
