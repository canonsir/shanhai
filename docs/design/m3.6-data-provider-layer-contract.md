## 0. 本设计的硬约束（不变量，继承 ADR 0021 §4/§5）

- **doc-only**：本文不写任何 `.py`，不建 `providers/{ifind,akshare,mock}` 目录，不建测试文件。
- **零 schema 改动**：复用现有 `knowledge_observation` spine 与 `Observation` DTO；不新增表、不改字段。
- **不接数据源**：iFinD/Wind/akshare/Tushare Pro 一律不接、不申请 token。
- **统一 Observation（ADR 0021 D3）**：provider DTO 永不泄漏到 Knowledge/Context；下游只认 `Observation`。
- **归一化集中（ADR 0021 D5）**：provider 吐草稿，`content_hash/logical_key` 由共享 ingestion pipeline 算。
- **6 条 Provider Boundary（ADR 0021 D7）**：只产 Observation / 不产 Knowledge / 不调 LLM / 不访问 Evolution / 可替换 / credential 配置化。
- **Review Gate**：本文完成后不写实现。实现拆步（M3.6.0→M3.6.4），每步经 Review Gate。

## 1. 概念地基：入口层在认知链路的位置

```
External Data Sources        iFinD / Wind / akshare / crawler（结构千差万别）
        │
        ▼  ← Data Provider Layer（本文契约层）：source-specific adapter
provider.fetch_observations(query) -> tuple[ObservationDraft, ...]
        │
        ▼  ← Observation Ingestion Pipeline（M3.6.2，source-neutral，算身份）
ObservationReadPort.record_many(observations) -> knowledge_observation spine（append-only）
        │
        ▼  ← 下游对 provider 无感知
Knowledge Evolution（ADR 0020）→ KnowledgeView（S4.3）→ ContextAssembler → MarketContextSnapshot
```

关键区分（**两个不同的东西，不要合并**）：

| | `ObservationDraft`（provider 产物） | `Observation`（落库 DTO，现存） |
|--|--|--|
| 归属 | Data Provider Layer 输出 | `knowledge_observation` spine 一行的只读投影 |
| 身份 | **无** `content_hash`/`logical_key`（未定身份） | **有** `logical_key + content_hash`（身份完整） |
| 谁产生 | provider adapter 的 mapper | ingestion pipeline 算身份后构造 / 读端口回放 |
| 定位 | 「取到的原始观测，尚未定身份」 | 「进入 spine 的确定性事实」 |
| 文件 | 新增（M3.6.0） | 复用 [observation_reader.py](../../services/market-data/shanhai_market_data/ports/observation_reader.py)（不改） |

> 为何不让 provider 直接产 `Observation`：见 ADR 0021 D5——身份规则若分散到 N 个 adapter，换 provider 时同一事实可能被判成两条 observation。集中到 pipeline 一处，换 provider 不影响身份逻辑。

## 2. M3.6.0 — Provider Contract（契约草图）

### 2.1 `DataQuery`（provider 的输入）

描述「要取谁、取什么家族、取什么时间范围」，**source-neutral**（不含任何 vendor 参数）：

```python
# 契约草图（非实现）—— 归属 services/market-data，复用 SubjectRef / FactType
class DataQuery(_FrozenModel):
    subject: SubjectRef                     # 取哪个主体（surrogate identity，非外部 code）
    fact_types: tuple[FactType, ...] = ()   # 取哪些家族；空 = provider 能力范围内全取
    since: datetime | None = None           # effective 下界（可选，世界视角）
    until: datetime | None = None           # effective 上界（可选）
    # 注：vendor 专属参数（ts_code / 频率 / 复权口径）不进 DataQuery，
    #     由各 adapter 在 mapper 内部从 subject/fact_types 推导，避免污染 source-neutral 契约。
```

### 2.2 `ObservationDraft`（provider 的产物，未定身份）

```python
# 契约草图（非实现）—— provider 唯一合法产物；刻意【不含】content_hash / logical_key
class ObservationDraft(_FrozenModel):
    subject: SubjectRef                     # 复用 market-data 主体引用
    fact_type: FactType                     # 复用 market-data 事实家族
    predicate: str | None = None            # 语义主张键（如 "pe_ttm"）
    object_value: str | None = None         # 值（字符串化，语义充分子集）
    occurred_at: datetime | None = None     # 世界视角（可选）
    published_at: datetime | None = None    # 世界视角（可选）
    captured_at: datetime                   # 系统视角捕获时刻（knowledge_at 轴，必填）
    source_ref: SourceRef                   # provenance；provider 只在此出现（provider=... / dataset=...）
    # 【无】content_hash / logical_key —— 身份由 ingestion pipeline 统一计算（D5）
```

### 2.3 `ObservationProvider`（canonical 写侧契约）

```python
# 契约草图（非实现）—— canonical；命名不复用 legacy MarketDataProvider（撞名，见 ADR 0021 D6）
@runtime_checkable
class ObservationProvider(Protocol):
    name: str                               # provider 标识（进 SourceRef.provider）

    def fetch_observations(self, query: DataQuery) -> tuple[ObservationDraft, ...]:
        """取数并归一成观测草稿。可对未覆盖的 fact_type raise NotImplementedError，
        由 ingestion / 编排层优雅降级（对齐现有 PublicMarketDataProvider 的降级约定）。"""
        ...
```

- 与 M3.2 `PublicMarketDataProvider` 的关系（ADR 0021 D6）：后者是**读侧 `*Record` 契约**，保留标 legacy、继续可用；`ObservationProvider` 是**写侧 canonical**，新 provider（mock/ifind/akshare）实现它。二者并存，逐步收敛。
- Boundary 落地（ADR 0021 D7）：`fetch_observations` 的返回类型天然只能是 `ObservationDraft`（B1）；Protocol 签名不含任何 Knowledge/LLM/Evolution 类型（B2/B3/B4）；provider 可替换（B5）；credential 经 config 注入而非签名（B6，见 §5）。

## 4. M3.6.2 — Observation Ingestion Pipeline（契约草图，身份归属处）

共享、source-neutral，是「草稿 → 落库 Observation」的唯一归一化处：

```python
# 契约草图（非实现）—— 身份/幂等/provenance 规则集中一处（ADR 0021 D5）
class ObservationIngestionPipeline:
    def __init__(self, sink: ObservationRecordPort) -> None:  # sink = 已有 record_many 的写端口
        self._sink = sink

    def ingest(self, drafts: tuple[ObservationDraft, ...]) -> IngestReport:
        observations = tuple(self._to_observation(d) for d in drafts)
        self._sink.record_many(observations)   # 幂等于 (logical_key, content_hash)，append-only
        return IngestReport(ingested=len(observations), ...)

    @staticmethod
    def _to_observation(draft: ObservationDraft) -> Observation:
        logical_key = _derive_logical_key(draft)      # 统一逻辑键规则（subject+fact_type+predicate+…）
        content_hash = _derive_content_hash(draft)     # 统一内容指纹（对语义字段做确定性 hash）
        return Observation(logical_key=logical_key, content_hash=content_hash, ...draft 其余字段...)
```

- **`record_many` 已存在**（[in_memory](../../services/market-data/shanhai_market_data/ports/in_memory_observation_reader.py) / [sqlite](../../services/market-data/shanhai_market_data/ports/sqlite_observation_reader.py) 两实现均有，幂等于 `(logical_key, content_hash)`，且身份由调用方传入）——pipeline 是身份计算的自然归属，**无需改 port、无需改 schema**。
- `_derive_logical_key` / `_derive_content_hash` 的**具体规则**在 M3.6.2 实现期定稿（须与 M3.3 spine 既有写入语义一致）；本文只冻结「身份在此计算、不在 provider 里算」。
- 幂等收益：同一事实被不同 provider 各取一次，只要归一到相同语义字段 → 相同 `content_hash` → spine 去重，天然支撑「换 provider 不产生重复事实」。

## 5. M3.6.x — 配置与目录布局（契约草图）

### 5.1 providers 目录布局（对齐用户 spec）

```
services/market-data/shanhai_market_data/providers/
├── __init__.py            # 现有：CninfoAnnouncementProvider / EastMoneyProvider（M3.2 legacy peers）
├── base.py                # 新增（M3.6.0）：ObservationProvider / DataQuery / ObservationDraft 契约
├── mock/
│   └── client.py          # M3.6.1：MockObservationProvider
├── ifind/                 # M3.6.3（本轮不建）
│   ├── client.py          #   iFinD SDK 调用（credential 配置化）
│   ├── mapper.py          #   iFinD payload → ObservationDraft（vendor DTO 到此为止，不外泄）
│   └── config.py          #   IFIND_APP_KEY / IFIND_SECRET 读取
└── akshare/               # M3.6.4（本轮不建）
    ├── client.py
    └── mapper.py
```

- **vendor DTO 只活在各 adapter 的 `mapper.py` 内**，出了 `fetch_observations` 就只有 `ObservationDraft`——这是 ADR 0021 D3「统一 Observation 而非统一 Provider」的物理落地。

### 5.2 配置装配（对齐 factory.py 现有风格）

```python
# 契约草图（非实现）—— 对齐 factory.py 的 SHANHAI_MARKET_STORE 装配风格
def default_observation_provider() -> ObservationProvider:
    name = os.getenv("SHANHAI_DATA_PROVIDER", "mock").lower()   # 免费默认：mock（无 token 即可跑）
    if name == "mock":
        return MockObservationProvider()
    if name == "akshare":
        return AkshareObservationProvider()                      # M3.6.4
    if name == "ifind":
        return IFinDObservationProvider(config=IFinDConfig.from_env())  # M3.6.3，credential 配置化
    raise ValueError(f"Unknown SHANHAI_DATA_PROVIDER: {name!r}")
```

```
免费模式（默认）    SHANHAI_DATA_PROVIDER=mock   或   =akshare
增强模式（可选）    SHANHAI_DATA_PROVIDER=ifind
                   IFIND_APP_KEY=...   IFIND_SECRET=...     （只经 env 注入；B6：不硬编码/不入库/不进日志）
多 provider（未来） 按 capability 配 primary/fallback（本轮只登记方向，不实现）：
                   market: { primary: ifind, fallback: [akshare] } / fundamental: { primary: ifind } / news: { primary: [ifind, crawler] }
```

- **Context 层对 `SHANHAI_DATA_PROVIDER` 无感知**（ADR 0019 R1-3）：换 provider = ingestion 多产生带 `SourceRef(provider=...)` 的 observation；认知层禁 `if source == "ifind"`。
- credential 只进 `providers/ifind/config.py` 的 `from_env()`，**不进** `DataQuery` / `ObservationDraft` / `SourceRef` 明文（B6）。

## 6. Provider Boundary 落地映射（ADR 0021 D7 → 本契约）

| # | Boundary | 本契约如何保证 |
|--|--|--|
| B1 | 只产 Observation | `fetch_observations` 返回类型只能是 `tuple[ObservationDraft, ...]`；无其他出口 |
| B2 | 不产 Knowledge | `providers/**` 禁 import `shanhai_market_intelligence`（AST 守护，实现期落地） |
| B3 | 不调 LLM | `providers/**` 禁 import 任何 model/LLM 客户端；`ObservationDraft` 无自由文本 summary 载体 |
| B4 | 不访问 Evolution | Protocol 签名不含 `EvolutionStore`/`KnowledgeResolver`；依赖方向 market-data 不反向 |
| B5 | 可替换 | `ObservationProvider` Protocol + factory 装配；换 provider 只改 `SHANHAI_DATA_PROVIDER` |
| B6 | credential 配置化 | `IFinDConfig.from_env()`；credential 不进签名/DTO/日志 |
