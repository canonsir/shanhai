## 0. 本文定位（为什么 M3.6.4 不是 EastMoney Adapter）

M3.6 已经连续冻结三层：

```text
ObservationProvider
      │
      ▼
ObservationDraft
      │
      ▼
EffectiveScopeStrategy → IdentityStrategy → ObservationMaterializer
      │
      ▼
Observation
      │
      ▼
record_many()
```

唯一还没冻结的是真实源进入 `ObservationDraft` 之前的边界：

```text
HTTP / SDK
      │
      ▼
Raw Response
      │
      ▼
ObservationMapper
      │
      ▼
ObservationDraft
```

如果 M3.6.4 直接叫 “EastMoney Adapter Design”，后面 CNInfo / Tushare / iFinD / Wind 会重复一遍设计。真正会重复、也真正需要冻结的是 **`map(raw) → ObservationDraft`**，不是 EastMoney 的 HTTP 细节。因此本阶段改名为：

> **M3.6.4 Observation Adapter Contract Design**

本文的核心裁决：

```text
Transport Policy 可以演进，
Mapper Contract 一旦冻结，应尽量保持稳定。
```

含义：

- HTTP / SDK / Retry / RateLimit / Pagination 是 **Transport/Resilience seam**：可以随源特性、网络策略、商业 SDK 演进。
- `Raw → ObservationDraft` 是 **Mapper Contract**：一旦冻结，应像 Provider Contract 与 Identity Pipeline 一样稳定；它决定不同数据源如何进入同一条 Observation Pipeline。

## 1. M1 — 两层契约，而不是一个“大 Adapter”

M3.6.4 冻结两层，不把机制和语义混在一起：

```text
┌─────────────────────────────────────────────────────────────┐
│ Transport / Resilience seam（可演进，adapter 内部机制）        │
│   HTTP / SDK / Auth / Retry / RateLimit / Pagination / JSON  │
│   产出：source-specific Raw Model                            │
└───────────────────────────────┬─────────────────────────────┘
                                │ Raw
                                ▼
┌─────────────────────────────────────────────────────────────┐
│ Mapper Contract（稳定，纯函数，跨源共享形态）                  │
│   source-specific Raw → source-neutral ObservationDraft       │
│   无 I/O / 无身份 / 无 dedup / 无 materialize / 无 Knowledge   │
└───────────────────────────────┬─────────────────────────────┘
                                │ ObservationDraft
                                ▼
                 ingest_drafts() / Identity Pipeline（M3.6.3）
```

### 1.1 Adapter MUST / MUST NOT

| Adapter 可做（MUST / MAY） | Adapter 禁止（MUST NOT） |
|--|--|
| 调 HTTP / SDK / 读取公开接口 | 计算 `logical_key` / `content_hash` |
| Auth / credential 装配（配置化） | dedup / conflict resolution |
| Retry / backoff / rate-limit handling | 调 `ObservationMaterializer` / `ingest_drafts` |
| Parse JSON / CSV / SDK object | 直接构造 `Observation` |
| 定义 source-specific Raw Model | 写 spine / 调 `record_many()` |
| `Raw → ObservationDraft` 机械映射 | 产 Belief / KnowledgeObject / KnowledgeRevision |
| 归一源表示（时间、编号、单位、字段名） | 调 LLM / ReasoningPort / Model Router |
| 构造 truthful `SourceRef` | import / 访问 market-intelligence / Evolution |

Adapter 的定位：

> Adapter 是事实搬运与源表示翻译层，不是身份层、认知层、冲突调和层、持久化层。

## 2. M2 — Raw Model（source-specific，不上提成共享模型）

不直接从任意 JSON dict map 到 draft。每个 adapter 内部必须先定义自己的 Raw Model，再交给 mapper：

```text
JSON / SDK object
      │ parse / validate source shape
      ▼
EastMoneyQuoteRaw / EastMoneyFinancialRaw / CninfoAnnouncementRaw / ...
      │ mapper
      ▼
ObservationDraft
```

### 2.1 Raw Model 的边界

| 项 | 冻结 |
|--|--|
| 所属位置 | source adapter 内部，如 `providers/eastmoney/raw.py`、`providers/cninfo/raw.py` |
| 性质 | source-specific；忠实表达该源返回形状 |
| 是否共享 | **不共享**；不得在 `shanhai_market_data` 顶层创建 `QuoteRaw` 这类伪共享模型 |
| 是否进入身份 | 不进入；Raw 只服务 mapper，不能越过 `ObservationDraft` |
| 测试方式 | Raw fixture → Raw Model → Mapper → Draft，离线测试 |

### 2.2 为什么 Raw Model 不放到共享层

`EastMoneyQuoteRaw` / `CninfoAnnouncementRaw` / `TushareFinancialRaw` 天然携带源字段名、接口版本、分页形态和缺字段习惯。如果把它们上提成共享模型，共享层会被某个源的 JSON 形状污染。

共享层只冻结：

- Mapper 接口形态；
- `ObservationDraft` 字段完备性规则；
- `SourceRef` 映射规则；
- Error / Pagination / Retry 的策略边界；
- Offline fixture 测试形态。

Raw 类型本身属于具体 adapter，M3.6.5+ 各源实现时再落。

## 3. M3 — Mapper Contract（Raw → ObservationDraft，纯函数）

Mapper 是 M3.6.4 真正要冻结的核心：

```python
# contract sketch —— 非实现
@runtime_checkable
class ObservationMapper(Protocol[RawT]):
    """source-specific Raw → source-neutral ObservationDraft。

    纯函数：无 HTTP、无 SDK 调用、无 retry、无 credential、无身份、无写库。
    一条 Raw 可以映射成 0..N 条 ObservationDraft：
    - quote raw 可能产 close/open/high/low/volume 等多条 draft
    - financial row 可能产 revenue/net_profit/roe 等多条 draft
    - announcement raw 通常产一条公告 draft
    """

    def map(self, raw: RawT, *, context: MappingContext) -> tuple[ObservationDraft, ...]:
        ...
```

`MappingContext` 是 mapper 的源中性输入上下文，不是身份上下文：

```python
# contract sketch —— 非实现
class MappingContext(_FrozenModel):
    subject: SubjectRef
    captured_at: datetime
    provider: str
    dataset: str
    adapter_version: str = "v1"
```

### 3.1 Mapper MUST / MUST NOT

| Mapper MUST | Mapper MUST NOT |
|--|--|
| 把源字段名映射为 source-neutral `predicate` | 发 HTTP / SDK |
| 把源时间映射为 `occurred_at` / `published_at` | Retry / sleep / rate limit |
| 把源唯一编号映射到 `SourceRef.external_id` | 算 `logical_key` / `content_hash` |
| 把数值/金额/比率统一成 `object_value: str` | 调 `CanonicalSerializer` |
| 构造 `SourceRef`（truthful provenance） | 调 `ObservationMaterializer` / `ingest_drafts` |
| 对缺失必填语义字段抛 mapper-level error | import market-intelligence / Knowledge / LLM |

### 3.2 与 M3.6.3 的硬接缝

M3.6.3 已冻结 `EffectiveScopeStrategy`。因此 mapper 必须保证它需要的字段已被源表示归一：

| fact_type | mapper 必须填给 draft 的字段 | 原因 |
|--|--|--|
| `QUOTE` | `occurred_at` = 交易日对应时间（至少可归一到 date） | `EffectiveScopeStrategy` 用它算 `trade_date` |
| `FINANCIAL` | `occurred_at` = 报告期末日 | 用它算 `report_period` |
| `ANNOUNCEMENT` | `source_ref.external_id` = 公告唯一号 | 用它算 `announcement_id`，非日期 |
| `NEWS` / `POLICY` | `published_at` 优先，否则 `occurred_at` | 用它算 `publish_period` |
| `PROFILE` | 可不填 `occurred_at` / `published_at` | latest-only，scope = `""` |

这条边界非常关键：

```text
Mapper 负责源表示归一；
EffectiveScopeStrategy 负责语义身份归一。
```

如果 mapper 没把公告 id 放进 `source_ref.external_id`，Identity Pipeline 就无法稳定算出 `announcement_id`；这不是 IdentityStrategy 的锅，是 mapper contract 违约。

## 4. M4 — ObservationDraft 完备性规则

`ObservationDraft` 已在 M3.6.2 冻结。本阶段只冻结 mapper 产出 draft 时的字段规则：

| 字段 | 规则 | 说明 |
|--|--|--|
| `subject` | MUST | 来自 `MappingContext.subject`，外部代码不得成为 ShanHai identity |
| `fact_type` | MUST | 必须用既有 `FactType`；不新增临时字符串类型 |
| `predicate` | SHOULD | 除 profile/公告标题等少数天然单语义外，应给稳定 predicate |
| `object_value` | SHOULD | 有值事实必须给字符串；结构化富字段不塞 draft 顶层 |
| `occurred_at` | fact_type dependent | QUOTE/FINANCIAL/INDUSTRY/ANOMALY/CAPITAL_FLOW/SHAREHOLDER 必填 |
| `published_at` | fact_type dependent | NEWS/POLICY/ANNOUNCEMENT 能获取则应填；NEWS/POLICY 优先依赖它 |
| `captured_at` | MUST | 来自 `MappingContext.captured_at`；同批 raw 应可共享 |
| `source_ref` | MUST | truthful provenance；见 §5 |

### 4.1 缺失字段策略

| 场景 | 策略 |
|--|--|
| 缺失可选展示字段（如 label、摘要、附件 URL） | 允许 `None` 或放入未来 typed detail，不阻断 draft |
| 缺失必填 identity 支撑字段（如 quote `occurred_at`、financial report period、announcement external_id） | mapper error；不得产出不稳定 draft |
| 源字段存在但单位/比例不明确 | mapper error；不得猜测 |
| 源字段为空但语义上代表“无数据” | 返回空 tuple 或 provider-level NotFound/NoData 语义，不能伪造 `object_value=""` |

## 5. M5 — SourceRef Mapping（Provenance 冻结）

M3.6.4 不改 `SourceRef`。实际字段以 [models.py](../../services/market-data/shanhai_market_data/models.py) 为准：

```python
class SourceRef(_FrozenModel):
    source_id: str
    source_name: str
    trust_level: SourceTrustLevel = SourceTrustLevel.PUBLIC_AGGREGATOR
    external_id: str | None = None
    captured_at: datetime = Field(default_factory=datetime.utcnow)
    provider: str | None = None
    dataset: str | None = None
    raw_snapshot_ref: str | None = None
    version: str = "v1"
    hash: str | None = None
```

### 5.1 字段规则

| 字段 | Adapter/Mapper 规则 |
|--|--|
| `source_id` | MUST；稳定源 id，如 `eastmoney` / `cninfo` / `tushare` / `ifind` |
| `source_name` | MUST；人读名称，如 `EastMoney` / `CNInfo` |
| `trust_level` | MUST；按源性质设置（官方披露源优先 `OFFICIAL`，公共聚合源 `PUBLIC_AGGREGATOR`，商业聚合源按后续裁决） |
| `external_id` | fact_type dependent；ANNOUNCEMENT MUST 为公告唯一号；其他类型可填源侧行 id / symbol / report id |
| `captured_at` | MUST；应与 draft `captured_at` 一致，表示本次采集时间 |
| `provider` | MUST；adapter canonical id |
| `dataset` | MUST；具体接口/数据集，如 `eastmoney.daily` / `eastmoney.f10_main_finance` / `cninfo.announcement` |
| `raw_snapshot_ref` | SHOULD；指向 raw snapshot locator（如 `raw://...`），不要求本阶段实现重存储 |
| `version` | MUST；adapter/parser 版本，默认 `v1` |
| `hash` | SHOULD；raw payload 或 normalized raw 的内容指纹，使用 `sha256:` 前缀惯例 |

### 5.2 关于 `source_ref.url`

当前 `SourceRef` **没有** `url` 字段，M3.6.4 不新增字段、不改 spine。源 URL 的归属：

- raw 文件/接口快照定位：`raw_snapshot_ref`；
- 公告 PDF / 文档 URL：属于公告语义明细，M3.6.4 只可在 raw model / future typed detail 中保留，不上提到 `SourceRef.url`；
- 若未来确需一等字段，另开 ADR 或 additive SourceRef 扩展，不混入本阶段。

## 6. M6 — Error Policy（Transport 与 Mapper 分层处理）

Provider 对外仍只有两种形态：

```text
fetch_observations(query) -> tuple[ObservationDraft, ...]
或 raise Exception
```

但 adapter 内部必须区分错误归属：

| 错误 | 层 | 策略 |
|--|--|--|
| HTTP 404 / 源声明无数据 | Transport | 对“确实无数据”的查询可返回空 tuple；对异常 URL/接口漂移应抛 source error |
| HTTP 5xx / network timeout | Transport | retry 后仍失败则抛 transient source error |
| Rate Limit / quota exceeded | Transport | 按 retry policy/backoff；仍失败抛 rate-limited source error |
| Auth / credential invalid | Transport | 抛 credential/config error；不得在 mapper 内处理 |
| JSON decode / SDK response shape invalid | Transport parse | 抛 decode/schema error；不得传 dict 给 mapper 猜 |
| Raw Model 校验失败 | Raw parse | 抛 raw schema error |
| 必填语义字段缺失 | Mapper | 抛 mapper contract error；不得产不稳定 draft |
| 可选字段缺失 | Mapper | `None` 或跳过对应 draft |

原则：

- Transport error 可以随策略演进；
- Mapper contract error 代表源数据无法满足 `ObservationDraft` 稳定语义，不能降级成错误 identity；
- Provider Registry / Composition Root 不负责解释源错误，只装配 provider。

## 7. M7 — Pagination Contract（Cursor → Page → Drafts）

很多公告/新闻/财务接口天然分页。M3.6.4 冻结分页语义，但不实现具体 paginator：

```text
Cursor
   │ transport fetch
   ▼
RawPage[RawT]
   │ mapper per raw item
   ▼
tuple[ObservationDraft, ...]
```

契约草图（非实现）：

```python
class PageCursor(_FrozenModel):
    value: str | None = None   # opaque，adapter 自己解释

class RawPage(_FrozenModel, Generic[RawT]):
    items: tuple[RawT, ...]
    next_cursor: PageCursor | None = None
    exhausted: bool = False
```

冻结规则：

- Cursor 是 opaque token；共享层不解释页码/offset/date window。
- Page 只承载 raw items 与下一页信息，不承载 `ObservationDraft`。
- Mapper 不负责翻页；它只 map 单条 raw 或单个 raw page 中的 item。
- Pagination loop 属 Transport/Resilience seam，可演进。
- 分页接口必须有终止条件：`exhausted=True` 或 `next_cursor is None`。

## 8. M8 — Retry / Transport Policy（可演进 seam）

Retry 属 transport，不属 mapper。M3.6.4 只冻结边界，不冻结具体 backoff 参数：

```text
TransportClient
   ├─ auth headers / SDK credential
   ├─ retry/backoff/rate-limit
   ├─ parse response bytes
   └─ produce Raw Model / RawPage
```

冻结规则：

- Retry 只包 transient transport errors（timeout / 5xx / rate-limit 可恢复场景）。
- Mapper error 不 retry；重试不会修复缺失语义字段。
- Auth/config error 不 retry；需要配置修复。
- TransportClient 必须可注入（对齐既有 `test_public_providers.py` 的 fake transport），保证 CI 离线。
- 默认 policy 可以保守；具体次数/backoff/jitter 属实现细节，可在 M3.6.5+ 演进。

## 9. M9 — Test Contract（Raw Fixture → Mapper → Draft）

M3.6.4 之后的 adapter 测试优先验证：

```text
Raw Fixture
   │ parse into Raw Model
   ▼
Mapper
   │
   ▼
ObservationDraft
```

而不是：

```text
HTTP
   │
   ▼
ObservationDraft
```

这样 CI 不依赖联网，不依赖商业 token，不被源限流影响。

### 9.1 必须覆盖的契约用例

| # | 用例 | 断言 |
|--|--|--|
| ① | Raw fixture → Raw Model | 能离线 parse source-shaped payload，不发网络 |
| ② | Raw Model → Draft | mapper 产出 `tuple[ObservationDraft, ...]` |
| ③ | Draft 无身份 | 不含 `logical_key` / `content_hash` / `confidence` / `observation_id` |
| ④ | EffectiveScope 支撑字段 | QUOTE 有 `occurred_at`；FINANCIAL 有 report period；ANNOUNCEMENT 有 `source_ref.external_id` |
| ⑤ | SourceRef truthfulness | provider/dataset/trust_level/external_id/version/hash/raw_snapshot_ref 按规则填 |
| ⑥ | Missing required field | 缺 identity 支撑字段时 mapper 抛错，不产不稳定 draft |
| ⑦ | Optional missing field | 可选字段缺失时允许 `None` 或跳过对应 draft |
| ⑧ | Pagination fixture | 多页 raw fixture 可合并成 drafts，终止条件明确 |
| ⑨ | Transport injected | fake transport 覆盖 HTTP path，测试不联网 |
| ⑩ | Boundary AST guard | adapter/mapper 不 import market-intelligence、不含 LLM/Knowledge/Materializer/record_many 符号 |

### 9.2 纸面压测（用现有 fixture）

现有 [test_public_providers.py](../../tests/market_data/test_public_providers.py) 已有真实形状 fixture：

- EastMoney push2 quote/profile raw：`_PUSH2`
- EastMoney F10 financial raw：`_F10`
- CNInfo topSearch raw：`_TOP_SEARCH`
- CNInfo paged announcement raw：`_HIS_ANN`
- fake transport seam：`_eastmoney_transport` / `_cninfo_transport`

M3.6.4 不改这些测试，但用它们证明契约足够覆盖：

| fixture | 未来 Raw Model | mapper 输出 draft 要点 |
|--|--|--|
| `_PUSH2` | `EastMoneyQuoteRaw` / `EastMoneyProfileRaw` | QUOTE drafts 必须有 `occurred_at`→trade_date；PROFILE scope 可为空 |
| `_F10` | `EastMoneyFinancialRaw` | FINANCIAL drafts 必须把 `REPORT_DATE` 归一为 report period |
| `_HIS_ANN` | `CninfoAnnouncementRaw` | ANNOUNCEMENT drafts 必须把 `announcementId` 放进 `source_ref.external_id` |

## 10. M10 — 命名与 coexistence

legacy M3.2 已存在：

- `EastMoneyProvider`
- `CninfoAnnouncementProvider`
- `PublicDataAcquisitionService`
- `PublicMarketDataProvider`

M3.6 canonical adapter 不能直接重名覆盖。冻结命名方向：

| 层 | 命名建议 |
|--|--|
| canonical provider | `EastMoneyObservationProvider` / `CninfoObservationProvider` |

| mapper | `EastMoneyObservationMapper` / `CninfoAnnouncementMapper` |
| transport client | `EastMoneyTransportClient` / `CninfoTransportClient` |

coexistence 规则：

- legacy provider 保留到迁移完成，不在 M3.6.4 修改；
- canonical provider 只实现 `ObservationProvider`，返回 `ObservationDraft`；
- 不再新增 legacy `fetch_quote/fetch_financial/fetch_announcement` 风格 provider；
- 退役 legacy 路线另走 Review Gate。
