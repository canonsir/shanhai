# ADR 0001：语言栈与运行时选择

状态：已采纳
日期：2026-06-22

## 背景

`services/` 与 `apps/api` 需要确定实现语言与运行时。Spec 中所有核心模块（Model Router、Agent、Wiki）的接口伪码均为 Python，且 AI / Agent 生态（模型 SDK、向量、知识抽取）以 Python 为主。`apps/web` 属于前端展示层。

## 决定

- `services/*`（harness-core、model-router、agent-runtime、wiki-engine、data-pipeline、evaluation）与 `apps/api` 使用 **Python 3.11+，API 框架 FastAPI**。
- `apps/web` 使用 **Node（前端工程）**。
- `packages/schemas`、`packages/prompts`、`packages/tools` 使用 Python。

## 原因

- Python 与 AI / Agent 生态契合，降低后续接入模型与知识抽取的成本。
- FastAPI 轻量、类型友好、与 Pydantic 协同好。
- Web 前端使用 Node 是行业惯例，独立演进。

## 影响

- 形成 Python（后端/服务）+ Node（前端）双运行时的 Monorepo。
- 跨语言契约通过 `packages/schemas` 与 OpenAPI 维持。
- Phase 0 `apps/web` 仅占位，不引入重前端框架。
